## Step 04 완결: Fp12 — 왜 12차까지 확장하는가

### 핵심 질문: 왜 하필 12?

페어링 `e(G1, G2) → GT`에서 GT는 **Fp^k의 부분군**이다. 이 k를 **embedding degree**라고 부른다.

```
embedding degree k란?
  → r | (p^k - 1) 을 만족하는 최소 k

  r = 커브의 위수 (group order)
  p = 기저체의 소수 (base field prime)

BN254의 경우:
  r | (p¹² - 1)  ← k = 12가 최소
  r ∤ (p¹ - 1), r ∤ (p² - 1), ..., r ∤ (p¹¹ - 1)
```

> [!important] embedding degree의 의미 페어링 결과가 "살 수 있는" 가장 작은 확장체가 Fp12라는 뜻이다.
> 
> - k가 너무 작으면 → DLP가 쉬워져서 보안 깨짐
> - k가 너무 크면 → 연산 비용 폭증
> - BN254의 k=12는 **128-bit 보안**과 **효율성**의 균형점

---

### 왜 직접 Fp12를 만들지 않고 타워를 쌓는가?

```
방법 A: Fp12 = Fp[x] / (x¹² - ...)
  → 곱셈: 12×12 = 144 Fp 곱셈 (naive)

방법 B: 타워 Fp → Fp2 → Fp6 → Fp12
  → 2 × 3 × 2 = 12 (차수 분해)
  → 각 층에서 Karatsuba 적용
  → 실제 Fp 곱셈 횟수가 크게 줄어듦
```

|층|확장|Karatsuba 절감|
|---|---|---|
|Fp → Fp2|2차, u²=-1|4회 → **3회**|
|Fp2 → Fp6|3차, v³=β|9회 → **6회**|
|Fp6 → Fp12|2차, w²=v|4회 → **3회**|

> [!tip] 3 × 6 × 3 = 54 Fp 곱셈 naive 144회 대비 **약 1/3**로 줄어든다. 이것이 타워의 가치.

---

### Fp6에 추가: mul_by_nonresidue (v 곱하기)

Fp12의 곱셈에서 w² = v가 등장하면, **Fp6 원소에 v를 곱하는** 연산이 필요하다.

```
v · (c₀ + c₁v + c₂v²) = c₀v + c₁v² + c₂v³
                        = c₂·β + c₀·v + c₁·v²
                          ^^^
                          v³ = β로 대체
```

**성분이 한 칸씩 회전**하고, 넘친 c₂에 β가 곱해진다:

```rust
// fp6.rs
/// v · (c₀ + c₁v + c₂v²) = c₂β + c₀v + c₁v²
pub fn mul_by_nonresidue(&self) -> Fp6 {
    Fp6 {
        c0: self.c2.mul_by_nonresidue(), // c₂ · β  (Fp2의 mul_by_nonresidue)
        c1: self.c0,                      // c₀ → v의 계수로
        c2: self.c1,                      // c₁ → v²의 계수로
    }
}
```

> [!note] non-residue 체인
> 
> ```
> Fp12 곱셈에서 w² = v 처리  →  Fp6::mul_by_nonresidue  (v 곱하기)
> Fp6 곱셈에서 v³ = β 처리   →  Fp2::mul_by_nonresidue  (β 곱하기)
> Fp2 곱셈에서 u² = -1 처리  →  그냥 부호 반전
> ```
> 
> 각 층의 "넘치는 항"을 처리하는 함수가 아래 층에 정의되어 있다.

### Fp12 구조체

```rust
// fp12.rs
use super::fp6::Fp6;

#[derive(Clone, Copy, PartialEq, Eq, Debug)]
pub struct Fp12 {
    pub c0: Fp6, // 상수항
    pub c1: Fp6, // w의 계수
}
```

```
Fp12 원소 = c₀ + c₁·w

  c₀, c₁ ∈ Fp6  (각각이 Fp2 3개 = Fp 6개)
  w는 새로운 미지수
  w² = v ∈ Fp6

총 차원: Fp6 × 2 = Fp2 × 6 = Fp × 12  ✓
```

### Fp12 곱셈: Karatsuba (Fp6 곱셈 3회)

Fp2와 **완전히 동일한 패턴**이다. 2차 확장이므로:

```
(a₀ + a₁w)(b₀ + b₁w) = a₀b₀ + a₁b₁·w² + (a₀b₁ + a₁b₀)w
                        = a₀b₀ + a₁b₁·v + (교차항)w
                                      ^^^
                                      w² = v로 대체
```

Karatsuba:

```
v₀ = a₀·b₀           ← Fp6 곱셈 ①
v₁ = a₁·b₁           ← Fp6 곱셈 ②

c₀ = v₀ + v₁·v       ← mul_by_nonresidue
c₁ = (a₀+a₁)(b₀+b₁) - v₀ - v₁   ← Fp6 곱셈 ③
```

```rust
pub fn mul(&self, rhs: &Fp12) -> Fp12 {
    let v0 = self.c0 * rhs.c0; // Fp6 곱셈 ①
    let v1 = self.c1 * rhs.c1; // Fp6 곱셈 ②

    // c₁ = (a₀+a₁)(b₀+b₁) - v₀ - v₁
    let c1 = (self.c0 + self.c1) * (rhs.c0 + rhs.c1) // ③
        - v0 - v1;

    // c₀ = v₀ + v₁·v  (w² = v이므로)
    let c0 = v0 + v1.mul_by_nonresidue();

    Fp12 { c0, c1 }
}
```

> [!tip] Fp2 곱셈과의 비교
> 
> ```
> Fp2:  c₀ = v₀ - v₁        ← u² = -1
> Fp12: c₀ = v₀ + v₁·v      ← w² = v (non-residue 곱)
> ```
> 
> 구조는 동일하고, "넘치는 항의 처리"만 다르다.

### Fp12 켤레 (conjugate)

```
conj(c₀ + c₁w) = c₀ - c₁w
```

Fp2의 켤레(`a₀+a₁u → a₀-a₁u`)와 **동일한 패턴**:

```rust
pub fn conjugate(&self) -> Fp12 {
    Fp12 {
        c0: self.c0,
        c1: -self.c1,
    }
}
```

> [!important] 페어링에서의 역할 final exponentiation의 "easy part"에서:
> 
> ```
> f^((p⁶-1)(p²+1)) 계산 시
> f^(p⁶) = conj(f)  ← Fp12에서 Frobenius^6 = conjugate
> ```
> 
> 즉, `f^(p⁶-1) = conj(f) · f⁻¹` 로 간단히 계산된다.

### Fp12 역원: Norm Reduction의 완성

```
norm = c₀² - v·c₁²  ∈ Fp6    ← Fp12 → Fp6로 내려감

a⁻¹ = conj(a) · norm⁻¹
     = (c₀ - c₁w) · norm⁻¹
```

```rust
pub fn inv(&self) -> Option<Fp12> {
    let c0s = self.c0.square();
    let c1s = self.c1.square();

    // norm = c₀² - v·c₁² ∈ Fp6
    let norm = c0s - c1s.mul_by_nonresidue();
    let norm_inv = norm.inv()?;  // Fp6의 inv 사용

    Some(Fp12 {
        c0: self.c0.mul(&norm_inv),
        c1: self.c1.neg().mul(&norm_inv),
    })
}
```

---

### 타워 전체의 inv 호출 체인

```
Fp12::inv()
  │  norm ∈ Fp6 계산
  │
  └─ Fp6::inv()
       │  norm ∈ Fp2 계산
       │
       └─ Fp2::inv()
            │  norm ∈ Fp 계산
            │
            └─ Fp::inv()  →  Fermat: a^(p-2)
```

> [!abstract] 패턴 요약 **모든 확장체의 역원은 같은 전략:**
> 
> |층|공식|norm이 사는 곳|
> |---|---|---|
> |Fp2|conj(a) / norm(a)|norm ∈ **Fp**|
> |Fp6|cofactor(a) / norm(a)|norm ∈ **Fp2**|
> |Fp12|conj(a) / norm(a)|norm ∈ **Fp6**|
> 
> 역원을 구할 때마다 **차원이 한 층씩 내려가서**, 최종적으로 Fp의 pow 하나로 귀결된다.

### 페어링과의 연결

```
e: G1 × G2 → GT ⊂ Fp12*

G1  — Fp 위의 점     (Step 05에서 구현)
G2  — Fp2 위의 점    (Step 05에서 구현)
GT  — Fp12의 부분군  (Step 06에서 사용)

페어링 계산 과정:
  1. Miller loop  → Fp12 원소 f를 누적 (mul, square 반복)
  2. Final exp    → f^((p¹²-1)/r) 계산 (inv, conjugate, frobenius 사용)
  3. 결과         → GT 원소
```

지금까지 만든 Fp/Fp2/Fp6/Fp12의 **모든 연산**이 페어링에서 사용된다:

|연산|사용처|
|---|---|
|Fp12::mul, square|Miller loop 누적|
|Fp12::conjugate|final exp easy part|
|Fp12::inv|final exp easy part|
|Fp12::frobenius|final exp hard part (다음에 구현)|
|Fp6::mul_by_nonresidue|Fp12 곱셈 내부|
|Fp2::mul_by_nonresidue|Fp6 곱셈 내부|
