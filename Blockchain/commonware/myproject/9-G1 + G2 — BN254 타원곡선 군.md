### 타원곡선이란?

```
유한체 위에서 y² = x³ + ax + b 를 만족하는 점 (x, y)의 집합 + 무한원점 O

BN254: a = 0, b = 3
  → y² = x³ + 3
```

> [!important] 왜 타원곡선인가? 점들이 **군(group)**을 이루고, `k·P`(스칼라 곱)는 쉽지만 `P`와 `Q = k·P`로부터 `k`를 구하는 것은 어렵다. → **이산로그 문제(ECDLP)** → 암호학적 안전성의 기반

---

### G1 vs G2: 왜 두 개?

```
페어링: e(G1, G2) → GT ⊂ Fp12*

G1 = Fp 위의 점    → 좌표가 Fp (256-bit 정수 1개)
G2 = Fp2 위의 점   → 좌표가 Fp2 (256-bit 정수 2개)
GT = Fp12의 부분군  → 페어링 결과
```

> [!note] 왜 G2는 Fp2 위에 있나? BN254의 embedding degree k=12에서, G2는 원래 E(Fp12) 위의 점이지만, **sextic twist**를 사용하면 E'(Fp2) 위의 점으로 표현 가능. 차원 12 → 2로 줄어들어 연산 비용이 극적으로 감소.

---

### Affine vs Jacobian 좌표

```
Affine (x, y):
  점 덧셈마다 역원(Fp::inv) 필요 — 254번의 square+mul → 비쌈!

Jacobian (X, Y, Z):
  affine (x, y) = (X/Z², Y/Z³)
  역원 없이 덧셈/더블링 가능
  최종 결과가 필요할 때만 to_affine으로 역원 1번
```

```rust
/// G1Affine — 외부 인터페이스용 (Fp 좌표)
pub struct G1Affine {
    pub x: Fp,
    pub y: Fp,
    pub infinity: bool,  // 무한원점 여부
}

/// G1 — 내부 연산용 Jacobian (Fp 좌표)
pub struct G1 {
    pub x: Fp,
    pub y: Fp,
    pub z: Fp,
}

/// G2Affine — 외부 인터페이스용 (Fp2 좌표)
pub struct G2Affine {
    pub x: Fp2,
    pub y: Fp2,
    pub infinity: bool,
}

/// G2 — 내부 연산용 Jacobian (Fp2 좌표)
pub struct G2 {
    pub x: Fp2,
    pub y: Fp2,
    pub z: Fp2,
}
```

> [!note] 왜 Affine과 Jacobian 두 타입인가?
> - **Jacobian** (`G1`, `G2`): 내부 연산용. 역원 없이 add/double 가능
> - **Affine** (`G1Affine`, `G2Affine`): 외부 인터페이스용. 직렬화, 비교, 페어링 입력
>
> 페어링 코드에서: `p.to_affine()` → Miller loop는 Affine 좌표로 line function 계산

> [!tip] Jacobian의 가치 페어링의 Miller loop에서 수백 번의 덧셈/더블링이 발생. 매번 inv를 피하는 것이 **극적인** 성능 차이를 만든다.

### G2: 트위스트 커브

#### 트위스트란?

```
원래: G2 ⊂ E(Fp12)  →  좌표마다 Fp 12개 필요
트위스트: G2 ⊂ E'(Fp2) →  좌표마다 Fp 2개 필요!

E': Y² = X³ + b'
b' = b/ξ = 3/(9+u)    ← D-type sextic twist
```

```rust
/// b' = 3/(9+u) — 런타임에 Fp2 산술로 계산
fn twist_b() -> Fp2 {
    let b = Fp2::new(Fp::from_u64(3), Fp::ZERO);
    let xi = Fp2::new(Fp::from_u64(9), Fp::from_u64(1));
    b * xi.inv().unwrap()
}
```

> [!abstract] 왜 ξ = 9+u인가? Fp6 = Fp2[v]/(v³-β)에서 β = 9+u. sextic twist의 twist 파라미터가 바로 이 β = ξ. 타워 구조의 non-residue가 twist에도 재활용된다.

### 점 더블링: 2P (G1/G2 공통)

```
a=0 특화 (BN254):

A = Y²
B = 4·X·A
C = 8·A²
D = 3·X²        ← a=0이므로 +aZ⁴ 없음

X₃ = D² - 2B
Y₃ = D·(B - X₃) - C
Z₃ = 2·Y·Z
```

```rust
// G1 버전 (Fp 산술)           // G2 버전 (Fp2 산술)
let a = self.y.square();        // 동일한 코드!
let b = four * self.x * a;     // 타입만 Fp → Fp2
let c = eight * a.square();
let d = three * self.x.square();
```

> [!tip] G1과 G2의 코드가 동일한 이유 Rust의 연산자 오버로딩 덕분에 **코드가 100% 동일**. `Fp::square()` 대신 `Fp2::square()`가 호출될 뿐. 이것이 타워 구조의 장점 — 상위 층이 하위 층의 산술을 재활용.

### 점 덧셈: P + Q

```
U₁ = X₁·Z₂²,  U₂ = X₂·Z₁²
S₁ = Y₁·Z₂³,  S₂ = Y₂·Z₁³
H = U₂ - U₁,   R = S₂ - S₁

엣지 케이스:
  U₁ == U₂ && S₁ == S₂ → P == Q  → double(P)
  U₁ == U₂ && S₁ != S₂ → P == -Q → identity

X₃ = R² - H³ - 2·U₁·H²
Y₃ = R·(U₁·H² - X₃) - S₁·H³
Z₃ = H·Z₁·Z₂
```

### 스칼라 곱: k·P (double-and-add)

```rust
pub fn scalar_mul(&self, scalar: &[u64; 4]) -> G1 {  // G2도 동일
    let mut result = Self::identity();
    let mut base = *self;
    for &limb in scalar.iter() {
        for j in 0..64 {
            if (limb >> j) & 1 == 1 {
                result = result.add(&base);
            }
            base = base.double();
        }
    }
    result
}
```

### 위수 검증: r·G = O

```rust
#[test]
fn scalar_mul_order_gives_identity() {
    let g = G2::generator();  // G1도 동일
    let r = [                  // BN254 group order
        0x43e1f593f0000001, 0x2833e84879b97091,
        0xb85045b68181585d, 0x30644e72e131a029,
    ];
    assert_eq!(g.scalar_mul(&r), G2::identity());
}
```

> [!abstract] G1과 G2의 위수가 같은 이유 둘 다 위수 r인 **소수위수 부분군**. 페어링 `e: G1 × G2 → GT`가 작동하려면 세 군의 위수가 모두 r이어야 한다.

### Jacobian 동치 비교

```
같은 Affine 점이 다른 Jacobian 표현을 가질 수 있다:
  (X, Y, Z) 과 (λ²X, λ³Y, λZ) 는 같은 점
```

```rust
impl PartialEq for G2 {
    fn eq(&self, other: &Self) -> bool {
        // to_affine 없이 비교: X₁Z₂² == X₂Z₁² and Y₁Z₂³ == Y₂Z₁³
        let z1s = self.z.square();
        let z2s = other.z.square();
        self.x * z2s == other.x * z1s
            && self.y * z2s * other.z == other.y * z1s * self.z
    }
}
```

### 좌표 변환: to_affine / to_projective

```
Jacobian → Affine: (X, Y, Z) → (X/Z², Y/Z³)
  → 역원(inv) 1회 필요 — 최종 결과가 필요할 때만 호출

Affine → Jacobian: (x, y) → (x, y, 1)
  → Z = 1로 두면 affine 좌표가 그대로 Jacobian
```

```rust
/// Jacobian → Affine: 역원 1회로 좌표 복원
pub fn to_affine(&self) -> G1Affine {  // G2도 동일
    if self.is_identity() {
        return G1Affine::identity();
    }
    let z_inv = self.z.inv().unwrap();
    let z_inv2 = z_inv.square();
    let z_inv3 = z_inv2 * z_inv;
    G1Affine {
        x: self.x * z_inv2,   // X/Z²
        y: self.y * z_inv3,   // Y/Z³
        infinity: false,
    }
}

/// Affine → Jacobian: Z = 1로 설정
pub fn to_projective(&self) -> G1 {  // G2Affine도 동일
    if self.infinity {
        G1::identity()
    } else {
        G1 { x: self.x, y: self.y, z: Fp::ONE }
    }
}
```

> [!tip] to_affine이 사용되는 곳
> - 페어링 함수 진입점: `p.to_affine()` → Miller loop에 전달
> - 테스트에서 좌표 비교: `g.to_affine().x == expected`
> - 직렬화/역직렬화

### 커브 위 검증: is_on_curve

```rust
// Affine: y² = x³ + b (G1) 또는 y² = x³ + b' (G2)
pub fn is_on_curve(&self) -> bool {
    if self.infinity { return true; }
    self.y.square() == self.x.square() * self.x + b
}

// Jacobian: Y² = X³ + b·Z⁶ (Z를 풀어야 함)
pub fn is_on_curve(&self) -> bool {
    if self.is_identity() { return true; }
    let z2 = self.z.square();
    let z6 = z2.square() * z2;  // Z⁴ · Z² = Z⁶
    self.y.square() == self.x.square() * self.x + b * z6
}
```

> [!note] Jacobian의 커브 방정식
> Affine에서 $y^2 = x^3 + b$이고, $x = X/Z^2$, $y = Y/Z^3$이므로:
> $(Y/Z^3)^2 = (X/Z^2)^3 + b$
> $Y^2/Z^6 = X^3/Z^6 + b$
> $Y^2 = X^3 + b \cdot Z^6$
