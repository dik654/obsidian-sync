## Step 11: QAP — m개의 제약을 하나의 다항식으로

### 핵심 질문: 왜 다항식으로 변환하는가?

```
R1CS (Step 09):
  m개의 개별 등식:
    ⟨a₁, s⟩·⟨b₁, s⟩ = ⟨c₁, s⟩
    ⟨a₂, s⟩·⟨b₂, s⟩ = ⟨c₂, s⟩
    ...
    ⟨aₘ, s⟩·⟨bₘ, s⟩ = ⟨cₘ, s⟩

  검증자가 모든 m개 등식을 일일이 확인해야 함 → O(m) 비용

QAP:
  m개의 등식을 "하나의 다항식 항등식"으로 압축:
    a(x)·b(x) - c(x) = h(x)·t(x)

  검증자는 랜덤 점 τ에서 한 번만 확인:
    a(τ)·b(τ) - c(τ) = h(τ)·t(τ)

  → O(1) 검증 (Groth16에서 페어링 3~4회)
```

> [!important] QAP = "m개의 이산 검사 → 1개의 다항식 항등식"
> ```
> 프로그램 → R1CS → [QAP] → Groth16
>                    ↑ 여기
>              "m개 → 1개"로 압축
>
> QAP가 없으면 Groth16의 O(1) 검증이 불가능하다.
> 이것이 ZK-SNARK의 "Succinct"(간결한)를 가능하게 하는 핵심.
> ```

### Schwartz-Zippel 보조정리: 왜 한 점이면 충분한가

```
보조정리 (Schwartz-Zippel):
  차수 d인 비영 다항식 p(x) ≠ 0이 있을 때,
  체 F에서 랜덤으로 선택한 τ에 대해:

    Pr[p(τ) = 0] ≤ d / |F|

BN254 스칼라체에서:
  d ≈ 1,000 (일반적인 회로의 제약 수)
  |F| ≈ 2^254

  → Pr[잘못된 증명이 통과할 확률] ≤ 1000 / 2^254 ≈ 10^(-74)
  → 사실상 0

직관:
  "차수 d인 다항식은 최대 d개의 근을 가진다"
  → 2^254개 중에서 d개만 특별
  → 랜덤으로 고른 점이 그 d개 중 하나일 확률 ≈ 0
```

> [!tip] 핵심 결론
> 랜덤 τ에서 `a(τ)·b(τ) - c(τ) = h(τ)·t(τ)`가 성립하면,
> "모든 x에서 성립한다"고 2^254 대 1의 확률로 확신할 수 있다.
> 이것이 하나의 점 검증으로 m개 제약을 동시에 검증하는 원리.

### R1CS에서 QAP로: 전체 그림

```
┌─────────────────────────────────────────────────────────────────┐
│                    R1CS → QAP 변환 파이프라인                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  R1CS 행렬 (A, B, C)                                            │
│  ┌─────────────┐                                                │
│  │ A: m×n 행렬  │  m = 제약 수                                    │
│  │ B: m×n 행렬  │  n = 변수 수                                    │
│  │ C: m×n 행렬  │                                                │
│  └──────┬──────┘                                                │
│         ↓                                                       │
│  ① 도메인 선택: D = {ω₁, ..., ωₘ}                                │
│         ↓                                                       │
│  ② 열별 Lagrange 보간                                            │
│     각 변수 j에 대해:                                             │
│       aⱼ(ωᵢ) = A[i,j]  →  aⱼ(x) 다항식                          │
│       bⱼ(ωᵢ) = B[i,j]  →  bⱼ(x) 다항식                          │
│       cⱼ(ωᵢ) = C[i,j]  →  cⱼ(x) 다항식                          │
│         ↓                                                       │
│  ③ 소거 다항식: t(x) = ∏(x - ωᵢ)                                 │
│         ↓                                                       │
│  ④ Witness 결합:                                                 │
│     a(x) = Σⱼ sⱼ·aⱼ(x)                                         │
│     b(x) = Σⱼ sⱼ·bⱼ(x)                                         │
│     c(x) = Σⱼ sⱼ·cⱼ(x)                                         │
│         ↓                                                       │
│  ⑤ h(x) = (a(x)·b(x) - c(x)) / t(x)                            │
│         ↓                                                       │
│  QAP 항등식: a(x)·b(x) - c(x) = h(x)·t(x)                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

> [!abstract] 핵심 동치
> ```
> R1CS 만족 ⟺ 모든 i에서 a(ωᵢ)·b(ωᵢ) = c(ωᵢ)
>            ⟺ a(x)·b(x) - c(x)가 모든 ωᵢ에서 0
>            ⟺ t(x) | (a(x)·b(x) - c(x))
>            ⟺ h(x) = (a(x)·b(x) - c(x)) / t(x)가 다항식 (나머지 없음)
> ```

### Part 1: 다항식 — 체 위의 기본 연산

```
다항식 표현: 계수 형태 (coefficient form)

  p(x) = a₀ + a₁x + a₂x² + ... + aₙxⁿ

  저장: coeffs = [a₀, a₁, a₂, ..., aₙ]
  차수: degree = n (= coeffs.len() - 1)

  예:
    p(x) = 3 + 2x + x²
    coeffs = [Fr(3), Fr(2), Fr(1)]
    degree = 2
```

```rust
/// 다항식: p(x) = coeffs[0] + coeffs[1]·x + coeffs[2]·x² + ...
///
/// 계수 표현 (coefficient form)
/// 교육용으로 O(n²) 연산 사용 — FFT 미적용
pub struct Polynomial {
    pub coeffs: Vec<Fr>,
}
```

#### Horner's method: O(n) 평가

```
나이브 평가: p(x) = a₀ + a₁x + a₂x² + a₃x³
  → x를 계산: x, x², x³         ← 곱셈 2회
  → 각 항: a₁·x, a₂·x², a₃·x³  ← 곱셈 3회
  → 합산: + + +                   ← 덧셈 3회
  → 총: 곱셈 5회, 덧셈 3회

Horner's method: p(x) = ((a₃·x + a₂)·x + a₁)·x + a₀
  → 안쪽부터:
    r = a₃                   ← 초기값
    r = r·x + a₂ = a₃x + a₂  ← 곱셈 1, 덧셈 1
    r = r·x + a₁              ← 곱셈 1, 덧셈 1
    r = r·x + a₀              ← 곱셈 1, 덧셈 1
  → 총: 곱셈 3회, 덧셈 3회 = O(n)

차이:
  나이브: O(n²) 곱셈 (x^k 계산에 k-1회)
  Horner: O(n) 곱셈 (정확히 n회)
```

> [!tip] 수동 추적: p(x) = 1 + 2x + 3x² 에서 x=4
> ```
> coeffs = [1, 2, 3]
>
> Horner (뒤에서 앞으로):
>   r = 0              (초기값)
>   r = 0·4 + 3 = 3    (coeffs[2])
>   r = 3·4 + 2 = 14   (coeffs[1])
>   r = 14·4 + 1 = 57  (coeffs[0])
>
> 검산: 1 + 2·4 + 3·16 = 1 + 8 + 48 = 57 ✓
> ```

```rust
/// 점 x에서 평가: Horner's method
pub fn eval(&self, x: Fr) -> Fr {
    if self.coeffs.is_empty() {
        return Fr::ZERO;
    }
    let mut result = Fr::ZERO;
    for &c in self.coeffs.iter().rev() {
        result = result * x + c;
    }
    result
}
```

#### 다항식 산술

```
덧셈: (a₀+a₁x) + (b₀+b₁x+b₂x²) = (a₀+b₀) + (a₁+b₁)x + b₂x²
  → 계수별 덧셈, 길이가 다르면 짧은 쪽을 0으로 패딩

뺄셈: 덧셈과 동일, 부호만 반전

곱셈 (schoolbook): O(n²)
  (a₀+a₁x)(b₀+b₁x) = a₀b₀ + (a₀b₁+a₁b₀)x + a₁b₁x²
  → coeffs[i+j] += a[i] * b[j]
  → 결과 차수 = deg(a) + deg(b)

스칼라 곱: s · p(x) = s·a₀ + s·a₁x + ...
  → 모든 계수에 s를 곱함
```

> [!note] 곱셈 복잡도
> ```
> 우리 구현: O(n²) schoolbook multiplication
> FFT 기반:  O(n log n) — NTT(Number Theoretic Transform) 사용
>
> 교육용으로는 O(n²)로 충분.
> 프로덕션(arkworks, bellman): roots of unity + NTT 사용.
> ```

### Part 2: Lagrange 보간 — QAP의 핵심 도구

```
문제:
  n개의 점 (x₀,y₀), (x₁,y₁), ..., (xₙ₋₁,yₙ₋₁)이 주어졌을 때
  p(xᵢ) = yᵢ를 만족하는 유일한 degree < n 다항식 p(x)를 구하라

해법: Lagrange 보간

  Step 1: 기저 다항식 Lᵢ(x) 정의
    Lᵢ(x) = ∏_{j≠i} (x - xⱼ) / (xᵢ - xⱼ)

    성질:
      Lᵢ(xᵢ) = 1  (분자 = 분모)
      Lᵢ(xⱼ) = 0  (j ≠ i, 분자에 (xⱼ-xⱼ)=0 포함)

  Step 2: 결합
    p(x) = Σᵢ yᵢ · Lᵢ(x)

    p(xₖ)에서: yₖ·1 + 나머지 0 = yₖ ✓

  유일성: degree < n인 다항식 중 유일 (대수학의 기본 정리)
```

#### 수동 추적: 2점 보간

```
점: (1, 3), (2, 5) → p(1)=3, p(2)=5인 degree ≤ 1 다항식

기저:
  L₀(x) = (x - 2)/(1 - 2) = (x - 2)/(-1) = -(x - 2) = -x + 2
  L₁(x) = (x - 1)/(2 - 1) = (x - 1)/1    = x - 1

검증:
  L₀(1) = -1 + 2 = 1,  L₀(2) = -2 + 2 = 0  ✓
  L₁(1) = 1 - 1 = 0,   L₁(2) = 2 - 1 = 1   ✓

결합:
  p(x) = 3·L₀(x) + 5·L₁(x)
       = 3(-x + 2) + 5(x - 1)
       = -3x + 6 + 5x - 5
       = 2x + 1

검증:
  p(1) = 2·1 + 1 = 3 ✓
  p(2) = 2·2 + 1 = 5 ✓
  p(0) = 1        (추가 점에서도 유효)
```

#### 수동 추적: 3점 보간 (x² 복원)

```
점: (1, 1), (2, 4), (3, 9)

기저:
  L₀(x) = (x-2)(x-3) / (1-2)(1-3) = (x-2)(x-3) / ((-1)(-2)) = (x-2)(x-3)/2
  L₁(x) = (x-1)(x-3) / (2-1)(2-3) = (x-1)(x-3) / ((1)(-1))  = -(x-1)(x-3)
  L₂(x) = (x-1)(x-2) / (3-1)(3-2) = (x-1)(x-2) / ((2)(1))   = (x-1)(x-2)/2

전개:
  L₀(x) = (x²-5x+6)/2
  L₁(x) = -(x²-4x+3) = -x²+4x-3
  L₂(x) = (x²-3x+2)/2

결합:
  p(x) = 1·L₀ + 4·L₁ + 9·L₂
       = (x²-5x+6)/2 + 4(-x²+4x-3) + 9(x²-3x+2)/2
       = x²/2 - 5x/2 + 3 - 4x² + 16x - 12 + 9x²/2 - 27x/2 + 9
       = (1/2 - 4 + 9/2)x² + (-5/2 + 16 - 27/2)x + (3 - 12 + 9)
       = (1/2 + 9/2 - 4)x² + (-5/2 - 27/2 + 16)x + 0
       = (10/2 - 4)x² + (-32/2 + 16)x
       = (5 - 4)x² + (-16 + 16)x
       = x²

검증: p(4) = 16 ✓   p(0) = 0 ✓
```

#### 구현: 기저 다항식 구축

```rust
pub fn lagrange_interpolate(points: &[(Fr, Fr)]) -> Self {
    let n = points.len();
    let mut result = vec![Fr::ZERO; n];

    for i in 0..n {
        let (xi, yi) = points[i];
        if yi.is_zero() { continue; }  // yᵢ=0이면 기여 없음

        // 분모: ∏_{j≠i} (xᵢ - xⱼ) — 스칼라
        let mut denom = Fr::ONE;
        for j in 0..n {
            if j != i { denom = denom * (xi - points[j].0); }
        }
        let denom_inv = denom.inv().expect("domain points must be distinct");

        // 분자: Lᵢ(x) = ∏_{j≠i} (x - xⱼ)
        // 다항식을 하나씩 곱하여 계수 벡터 구축
        let mut basis = vec![Fr::ZERO; n];
        basis[0] = Fr::ONE;  // 시작: 상수 1
        let mut deg = 0;
        for j in 0..n {
            if j == i { continue; }
            let xj = points[j].0;
            // basis *= (x - xⱼ)
            for k in (1..=deg+1).rev() {
                basis[k] = basis[k-1] - basis[k] * xj;
            }
            basis[0] = -basis[0] * xj;
            deg += 1;
        }

        // result += yᵢ · denom_inv · basis
        let coeff = yi * denom_inv;
        for k in 0..n { result[k] = result[k] + coeff * basis[k]; }
    }

    Polynomial::from_coeffs(result)
}
```

> [!important] 복잡도 분석
> ```
> Lagrange 보간: O(n²)
>   n개의 기저 다항식, 각 기저 구축에 O(n) 곱셈
>
> FFT 기반 보간: O(n log² n)
>   roots of unity 사용 시 NTT로 가속
>   우리 도메인 {1,2,...,m}은 roots of unity가 아니므로 FFT 불가
>
> 프로덕션 선택:
>   도메인 = {ω, ω², ..., ω^m} (m차 roots of unity)
>   → NTT로 O(n log n) 보간 + 평가
>   → 우리 코드에서 domain: Vec<Fr> = (1..=m).map(Fr::from_u64) 대신
>     ω^i를 사용
> ```

#### 기저 다항식 계수 구축의 상세 수동 추적

> [!abstract] 이 섹션의 목적
> `lagrange_interpolate`에서 가장 이해하기 어려운 부분은 **기저 다항식의 계수를 점진적으로 구축하는 내부 루프**이다.
> `basis *= (x - xⱼ)`를 계수 벡터 레벨에서 어떻게 수행하는지를 한 단계씩 추적한다.

```
문제: 다항식 p(x)에 (x - c)를 곱하면 계수가 어떻게 변하는가?

p(x) = a₀ + a₁x + a₂x² 일 때:

  p(x) · (x - c) = (a₀ + a₁x + a₂x²)(x - c)
                  = a₀x + a₁x² + a₂x³ - c·a₀ - c·a₁x - c·a₂x²

  정리:
    x⁰ 계수: -c·a₀
    x¹ 계수: a₀ - c·a₁
    x² 계수: a₁ - c·a₂
    x³ 계수: a₂

  일반화:
    new[0] = -c · old[0]
    new[k] = old[k-1] - c · old[k]   (k ≥ 1)
    new[n+1] = old[n]                 (최고차항)

이것이 코드의 내부 루프가 하는 일:
    for k in (1..=deg+1).rev() {
        basis[k] = basis[k-1] - basis[k] * xj;
    }
    basis[0] = -basis[0] * xj;
```

```
수동 추적: 3점 보간 (1,1), (2,4), (3,9)에서 i=0의 기저 L₀(x) 구축

L₀(x) = (x - x₁)(x - x₂) / ((x₀ - x₁)(x₀ - x₂))

분자 (x - 2)(x - 3)의 계수를 코드가 구축하는 과정:

━━━ 초기 상태 ━━━
  basis = [1, 0, 0]   ← 상수 다항식 p(x) = 1
  deg = 0

━━━ j=1 (j ≠ i=0): (x - x₁) = (x - 2) 곱셈 ━━━
  xj = 2

  루프: k = 1 (= deg+1 = 0+1), 역순
    basis[1] = basis[0] - basis[1] * 2
             = 1 - 0 * 2
             = 1

  basis[0] = -basis[0] * 2
           = -1 * 2
           = -2

  deg = 1

  basis 상태: [-2, 1, 0]
  → p(x) = -2 + x = (x - 2) ✓

━━━ j=2 (j ≠ i=0): (x - x₂) = (x - 3) 곱셈 ━━━
  xj = 3

  루프: k = 2 (= deg+1 = 1+1), k = 1, 역순
    k=2: basis[2] = basis[1] - basis[2] * 3
                   = 1 - 0 * 3
                   = 1

    k=1: basis[1] = basis[0] - basis[1] * 3
                   = -2 - 1 * 3
                   = -5

  basis[0] = -basis[0] * 3
           = -(-2) * 3
           = 6

  deg = 2

  basis 상태: [6, -5, 1]
  → p(x) = 6 - 5x + x² = (x-2)(x-3) ✓

  검증:
    p(2) = 6 - 10 + 4 = 0 ✓
    p(3) = 6 - 15 + 9 = 0 ✓
    p(1) = 6 - 5 + 1 = 2  (분모 값)

━━━ 분모 계산 ━━━
  denom = (x₀ - x₁)(x₀ - x₂) = (1-2)(1-3) = (-1)(-2) = 2
  denom_inv = 2⁻¹ mod r  (Fr에서의 역원)

━━━ 최종 기여 ━━━
  coeff = y₀ · denom_inv = 1 · (1/2)

  result에 추가:
    result[0] += coeff · basis[0] = (1/2) · 6 = 3
    result[1] += coeff · basis[1] = (1/2) · (-5) = -5/2
    result[2] += coeff · basis[2] = (1/2) · 1 = 1/2

  → y₀ · L₀(x) = (6 - 5x + x²) / 2 = 3 - (5/2)x + (1/2)x²
```

```
왜 역순(rev()) 루프인가?

  순방향으로 하면:
    k=0: basis[0] = -basis[0] * xj    ← a₀가 변경됨
    k=1: basis[1] = basis[0] - ...     ← 이미 변경된 a₀을 읽음! (잘못!)

  역순으로 하면:
    k=2: basis[2] = basis[1] - ...     ← 아직 변경 안 된 a₁을 읽음 ✓
    k=1: basis[1] = basis[0] - ...     ← 아직 변경 안 된 a₀을 읽음 ✓
    마지막: basis[0] = -basis[0] * xj  ← basis[0]은 마지막에 처리

  이것은 in-place 배열 갱신의 고전적 패턴:
    "높은 인덱스부터 처리하면 낮은 인덱스의 원본 값을 보존"

  비유:
    계단을 위에서부터 청소하는 것과 같다.
    아래에서 시작하면 올라갈 때 이미 청소한 곳을 밟게 된다.
```

```
역순 루프의 동작을 시각화 (deg=1 → deg=2 전환):

  basis = [-2, 1, 0],  xj = 3

  ┌─────────────────────────────────────────────────────────────┐
  │  Before:  basis = [-2,  1,  0]                               │
  │                     a₀  a₁  a₂                               │
  │                                                               │
  │  k=2: basis[2] = basis[1] - basis[2]·3                       │
  │                = 1 - 0·3 = 1                                 │
  │         basis = [-2,  1,  1]                                  │
  │                  a₀  a₁  ↑new                                │
  │                                                               │
  │  k=1: basis[1] = basis[0] - basis[1]·3                       │
  │                = -2 - 1·3 = -5                               │
  │         basis = [-2, -5,  1]                                  │
  │                  a₀  ↑new                                    │
  │                                                               │
  │  k=0: basis[0] = -basis[0]·3                                 │
  │                = -(-2)·3 = 6                                 │
  │         basis = [ 6, -5,  1]                                  │
  │                  ↑new                                         │
  │                                                               │
  │  After:  basis = [6, -5, 1]  →  6 - 5x + x²                 │
  │                                = (x-2)(x-3) ✓                │
  └─────────────────────────────────────────────────────────────┘

  핵심: k=1에서 basis[0] = -2 (원본)을 읽음
        만약 순방향이었으면 basis[0]이 이미 6으로 바뀌어 있을 것!
```

### Lagrange 기저 다항식의 시각화

> [!abstract] Lagrange 기저의 핵심 직관
> Lagrange 기저 다항식 L₀, L₁, L₂는 "선택자(selector)" 역할을 한다.
> 각 기저 다항식은 **정확히 하나의 도메인 점에서만 1**이고 나머지에서는 0이다.
> 이것은 마치 디지털 회로의 멀티플렉서처럼, 한 번에 하나의 제약만 "활성화"한다.

#### 도메인 {1, 2, 3}에서의 세 기저 다항식

```
복습: 도메인 D = {ω₁=1, ω₂=2, ω₃=3}

L₀(x) = (x-2)(x-3) / ((1-2)(1-3)) = (x-2)(x-3) / 2
L₁(x) = (x-1)(x-3) / ((2-1)(2-3)) = -(x-1)(x-3)
L₂(x) = (x-1)(x-2) / ((3-1)(3-2)) = (x-1)(x-2) / 2

계수 형태:
  L₀(x) = (x²-5x+6)/2    = (1/2)x² - (5/2)x + 3
  L₁(x) = -(x²-4x+3)     = -x² + 4x - 3
  L₂(x) = (x²-3x+2)/2    = (1/2)x² - (3/2)x + 1
```

#### 각 기저 다항식의 값 시각화

```
L₀(x): x=1에서만 1, 나머지에서 0

  값 │
   1 │    *
     │   / \
     │  /   \
   0 │-+-----*-----*-------→ x
     │ 0     1     2     3
  -1 │
     └─────────────────────

  L₀(1) = (1-2)(1-3)/2 = (-1)(-2)/2 = 2/2 = 1  ✓
  L₀(2) = (2-2)(2-3)/2 = (0)(-1)/2  = 0         ✓
  L₀(3) = (3-2)(3-3)/2 = (1)(0)/2   = 0         ✓


L₁(x): x=2에서만 1, 나머지에서 0

  값 │
   1 │          *
     │         / \
     │        /   \
   0 │--*----+-----*-------→ x
     │  0    1     2     3
  -1 │
     └─────────────────────

  L₁(1) = -(1-1)(1-3) = -(0)(-2) = 0             ✓
  L₁(2) = -(2-1)(2-3) = -(1)(-1) = 1             ✓
  L₁(3) = -(3-1)(3-3) = -(2)(0)  = 0             ✓


L₂(x): x=3에서만 1, 나머지에서 0

  값 │
   1 │                *
     │               / \
     │              /   \
   0 │--*-----*----+-------→ x
     │  0     1    2     3
  -1 │
     └─────────────────────

  L₂(1) = (1-1)(1-2)/2 = (0)(-1)/2 = 0           ✓
  L₂(2) = (2-1)(2-2)/2 = (1)(0)/2  = 0           ✓
  L₂(3) = (3-1)(3-2)/2 = (2)(1)/2  = 1           ✓
```

#### 전체 평가 테이블: Kronecker delta 성질

```
┌──────────┬──────────┬──────────┬──────────┐
│          │  x = 1   │  x = 2   │  x = 3   │
│          │  (ω₁)    │  (ω₂)    │  (ω₃)    │
├──────────┼──────────┼──────────┼──────────┤
│  L₀(x)   │    1     │    0     │    0     │
├──────────┼──────────┼──────────┼──────────┤
│  L₁(x)   │    0     │    1     │    0     │
├──────────┼──────────┼──────────┼──────────┤
│  L₂(x)   │    0     │    0     │    1     │
└──────────┴──────────┴──────────┴──────────┘

이것이 Kronecker delta 성질:

  Lᵢ(ωⱼ) = δᵢⱼ = { 1  if i = j
                   { 0  if i ≠ j

이 테이블은 단위 행렬(identity matrix)과 동일한 구조:

  ┌       ┐     ┌       ┐
  │ 1 0 0 │     │ L₀(ω₁) L₀(ω₂) L₀(ω₃) │
  │ 0 1 0 │  =  │ L₁(ω₁) L₁(ω₂) L₁(ω₃) │
  │ 0 0 1 │     │ L₂(ω₁) L₂(ω₂) L₂(ω₃) │
  └       ┘     └                          ┘
```

#### "스위치" 비유: 왜 이것이 QAP에서 중요한가

```
R1CS에는 m=3개의 제약이 있다:
  제약 0: x·x = t₁      (ω₁=1에서 인코딩)
  제약 1: t₁·x = t₂     (ω₂=2에서 인코딩)
  제약 2: (t₂+x+5)·1=y  (ω₃=3에서 인코딩)

Lagrange 기저는 각 제약에 대한 "스위치":

  ┌───────────────────────────────────────────────────────┐
  │  p(x) = y₀·L₀(x) + y₁·L₁(x) + y₂·L₂(x)             │
  │                                                        │
  │  x = ω₁ = 1 일 때:                                     │
  │    L₀ = ON (=1),  L₁ = OFF (=0),  L₂ = OFF (=0)       │
  │    → p(1) = y₀·1 + y₁·0 + y₂·0 = y₀                  │
  │    → 제약 0의 값만 "활성화"                               │
  │                                                        │
  │  x = ω₂ = 2 일 때:                                     │
  │    L₀ = OFF,  L₁ = ON,  L₂ = OFF                      │
  │    → p(2) = y₁                                         │
  │    → 제약 1의 값만 "활성화"                               │
  │                                                        │
  │  x = ω₃ = 3 일 때:                                     │
  │    L₀ = OFF,  L₁ = OFF,  L₂ = ON                      │
  │    → p(3) = y₂                                         │
  │    → 제약 2의 값만 "활성화"                               │
  └───────────────────────────────────────────────────────┘

비유:
  3개의 스위치가 달린 컨트롤 패널:
    x=1 → 스위치 0만 ON → 제약 0 검사
    x=2 → 스위치 1만 ON → 제약 1 검사
    x=3 → 스위치 2만 ON → 제약 2 검사

  하나의 연속 함수(다항식)가
  이산적인 제약들을 "인코딩"하는 메커니즘이다.
```

> [!note] Lagrange 기저와 R1CS 열 보간의 연결
> ```
> R1CS 행렬의 열 j를 보간할 때:
>   aⱼ(x) = A[0,j]·L₀(x) + A[1,j]·L₁(x) + A[2,j]·L₂(x)
>
> 예: 열 2 (변수 x)의 A 다항식
>   aⱼ(x) = 1·L₀(x) + 0·L₁(x) + 1·L₂(x)
>          = L₀(x) + L₂(x)
>
>   검증:
>     aⱼ(1) = L₀(1) + L₂(1) = 1 + 0 = 1  = A[0,2] ✓
>     aⱼ(2) = L₀(2) + L₂(2) = 0 + 0 = 0  = A[1,2] ✓
>     aⱼ(3) = L₀(3) + L₂(3) = 0 + 1 = 1  = A[2,2] ✓
>
> 즉, Lagrange 기저는 R1CS 행렬의 각 행을 "선택"하는 도구이다.
> x=ωᵢ 에서 평가하면 i번째 행의 값만 꺼내진다.
> ```

#### 기저 다항식의 합: 분할 단위원 성질

```
흥미로운 성질: L₀(x) + L₁(x) + L₂(x) = 1  (항등적으로!)

검증:
  L₀(x) + L₁(x) + L₂(x)
  = (1/2)x² - (5/2)x + 3 + (-x² + 4x - 3) + (1/2)x² - (3/2)x + 1
  = (1/2 - 1 + 1/2)x² + (-5/2 + 4 - 3/2)x + (3 - 3 + 1)
  = 0·x² + 0·x + 1
  = 1  ✓

이것은 "partition of unity" (분할 단위원) 성질이라 한다.

직관: 모든 "스위치"의 영향력을 합하면 항상 1이다.
      어떤 점 x에서든, 세 기저의 "가중치"를 합하면 정확히 1.
      → 보간된 다항식이 입력 값들의 "가중 평균"처럼 작동한다.
```

### Part 3: 다항식 나눗셈 — h(x) 계산의 핵심

```
다항식 long division:
  dividend = quotient · divisor + remainder

  dividend(x) / divisor(x)를 계산하면:
    quotient(x): 몫 다항식
    remainder(x): 나머지 다항식

  나누어떨어짐 = remainder가 0

QAP에서의 역할:
  h(x) = (a(x)·b(x) - c(x)) / t(x)

  valid witness → remainder = 0 → h(x) 존재 → QAP 만족
  invalid witness → remainder ≠ 0 → h(x) 부존재 → QAP 불만족
```

#### 수동 추적: (x² + 3x + 2) / (x + 1)

```
dividend: x² + 3x + 2
divisor:  x + 1

Step 1: x² / x = x (최고차 항)
  x · (x + 1) = x² + x
  remainder: (x² + 3x + 2) - (x² + x) = 2x + 2

Step 2: 2x / x = 2
  2 · (x + 1) = 2x + 2
  remainder: (2x + 2) - (2x + 2) = 0

결과: quotient = x + 2, remainder = 0

검산: (x + 2)(x + 1) = x² + 3x + 2 ✓
```

#### 수동 추적: (x² + 1) / (x + 1) — 나머지 있는 경우

```
dividend: x² + 0x + 1
divisor:  x + 1

Step 1: x² / x = x
  x · (x + 1) = x² + x
  remainder: (x² + 1) - (x² + x) = -x + 1

Step 2: -x / x = -1
  -1 · (x + 1) = -x - 1
  remainder: (-x + 1) - (-x - 1) = 2

결과: quotient = x - 1, remainder = 2

검산: (x - 1)(x + 1) + 2 = x² - 1 + 2 = x² + 1 ✓

나머지가 0이 아님 → "나누어떨어지지 않음"
→ QAP에서 이것은 "잘못된 witness"를 의미
```

```rust
pub fn div_rem(&self, divisor: &Self) -> (Self, Self) {
    // ...
    let mut remainder = self.coeffs.clone();
    let divisor_lead_inv = divisor.coeffs.last().unwrap().inv()
        .expect("leading coefficient must be non-zero");
    let div_deg = divisor.degree();
    let quot_deg = self.degree() - div_deg;

    let mut quotient = vec![Fr::ZERO; quot_deg + 1];
    for i in (0..=quot_deg).rev() {
        let coeff = remainder[i + div_deg] * divisor_lead_inv;
        quotient[i] = coeff;
        for j in 0..=div_deg {
            remainder[i + j] = remainder[i + j] - coeff * divisor.coeffs[j];
        }
    }
    (Polynomial::from_coeffs(quotient), Polynomial::from_coeffs(remainder))
}
```

### Part 4: 소거 다항식 (Vanishing Polynomial)

```
정의:
  도메인 D = {ω₁, ..., ωₘ}에 대한 소거 다항식:
    t(x) = (x - ω₁)(x - ω₂)···(x - ωₘ)

  성질:
    t(ωᵢ) = 0  ∀i  (도메인의 모든 점에서 0)
    t(x) ≠ 0   x ∉ D  (도메인 밖에서는 0이 아님)
    degree(t) = m

  역할:
    "다항식 p(x)가 모든 ωᵢ에서 0이다"
    ⟺ "t(x) | p(x)" (t(x)로 나누어떨어진다)

    이유: p(x)가 ω₁, ..., ωₘ에서 0이면
          (x-ω₁), ..., (x-ωₘ)이 모두 p(x)의 인수
          → ∏(x-ωᵢ) = t(x)가 p(x)의 인수
```

#### 수동 추적: 도메인 {1, 2, 3}

```
t(x) = (x - 1)(x - 2)(x - 3)

전개:
  (x - 1)(x - 2) = x² - 3x + 2
  (x² - 3x + 2)(x - 3) = x³ - 3x² - 3x² + 9x + 2x - 6
                        = x³ - 6x² + 11x - 6

coeffs = [-6, 11, -6, 1]

검증:
  t(1) = 1 - 6 + 11 - 6 = 0  ✓
  t(2) = 8 - 24 + 22 - 6 = 0  ✓
  t(3) = 27 - 54 + 33 - 6 = 0  ✓
  t(4) = 64 - 96 + 44 - 6 = 6  ≠ 0 ✓ (도메인 밖)
  t(0) = -6  ≠ 0 ✓ (도메인 밖)
```

```rust
pub fn vanishing_polynomial(domain: &[Fr]) -> Polynomial {
    let mut result = Polynomial::from_coeffs(vec![Fr::ONE]); // 1
    for &omega in domain {
        let factor = Polynomial::from_coeffs(vec![-omega, Fr::ONE]); // (x - ω)
        result = &result * &factor;
    }
    result
}
```

> [!note] 프로덕션에서의 소거 다항식
> ```
> 우리: t(x) = (x-1)(x-2)···(x-m)
>   → 계수가 큰 정수 → 하나씩 곱하여 O(m²) 구축
>
> 프로덕션: roots of unity 사용
>   ω = primitive m-th root of unity (ω^m = 1)
>   도메인 = {1, ω, ω², ..., ω^(m-1)}
>   → t(x) = x^m - 1 (!)
>
>   이유: ∏(x - ω^i) = x^m - 1 (roots of unity의 성질)
>   → 계수가 {0, 0, ..., 0, -1, 0, ..., 0, 1} — 극도로 희소
>   → 평가: t(τ) = τ^m - 1 — O(log m) 거듭제곱
>   → O(1) 저장, O(log m) 평가
>
> 이것이 roots of unity가 QAP에서 중요한 이유.
> ```

### Part 5: R1CS → QAP 변환 — 열 보간

```
변환 과정:

  입력: R1CS 행렬 A, B, C (m×n), 도메인 D = {ω₁, ..., ωₘ}

  각 변수 j (= 행렬의 열)에 대해:
    열 A[*,j]의 값: A[0,j], A[1,j], ..., A[m-1,j]
    → 다항식 aⱼ(x) 보간: aⱼ(ωᵢ) = A[i,j]

    마찬가지로 bⱼ(x), cⱼ(x) 보간

  핵심:
    대부분의 열은 거의 0 (희소 행렬)
    → 모든 원소가 0인 열: 보간 생략 → zero polynomial

  출력:
    aⱼ(x), bⱼ(x), cⱼ(x) — 각 변수에 대한 다항식 (n개씩)
    t(x) — 소거 다항식
```

```rust
fn columns_to_polynomials(
    matrix: &[Vec<(usize, Fr)>],  // 행별 희소 표현
    num_vars: usize,
    domain: &[Fr],
) -> Vec<Polynomial> {
    let m = matrix.len();

    (0..num_vars).map(|j| {
        // 열 j의 모든 값 수집
        let mut all_zero = true;
        let points: Vec<(Fr, Fr)> = (0..m).map(|i| {
            let val = matrix[i].iter()
                .find(|&&(col, _)| col == j)
                .map(|&(_, v)| v)
                .unwrap_or(Fr::ZERO);
            if !val.is_zero() { all_zero = false; }
            (domain[i], val)
        }).collect();

        if all_zero {
            Polynomial::zero()  // 최적화: 0열은 보간 생략
        } else {
            Polynomial::lagrange_interpolate(&points)
        }
    }).collect()
}
```

### Part 6: Witness 결합과 h(x) 계산

```
Witness 결합:
  witness 벡터 s = [s₀, s₁, ..., sₙ₋₁]로 열 다항식들을 결합:

  a(x) = Σⱼ sⱼ · aⱼ(x)    (n개의 다항식을 가중합)
  b(x) = Σⱼ sⱼ · bⱼ(x)
  c(x) = Σⱼ sⱼ · cⱼ(x)

핵심 성질:
  도메인 점 ωᵢ에서:
    a(ωᵢ) = Σⱼ sⱼ · aⱼ(ωᵢ) = Σⱼ sⱼ · A[i,j] = ⟨A[i], s⟩
    b(ωᵢ) = ⟨B[i], s⟩
    c(ωᵢ) = ⟨C[i], s⟩

  따라서:
    a(ωᵢ) · b(ωᵢ) = ⟨A[i],s⟩ · ⟨B[i],s⟩
    c(ωᵢ) = ⟨C[i],s⟩

  R1CS 만족 ⟺ 모든 i에서 위 두 값이 같음
            ⟺ a(x)·b(x) - c(x)가 모든 ωᵢ에서 0
            ⟺ t(x)로 나누어떨어짐
```

```
h(x) 계산:
  p(x) = a(x)·b(x) - c(x)

  valid witness:
    p(ωᵢ) = 0 for all i
    → p(x) = h(x)·t(x)  (정확히 나누어떨어짐)
    → h(x)는 차수 ≤ m-2인 다항식

  invalid witness:
    ∃i: p(ωᵢ) ≠ 0
    → p(x)는 t(x)로 나누어떨어지지 않음
    → 나머지 ≠ 0
    → h(x) "존재하지 않음"
```

```rust
pub fn compute_h(&self, witness: &[Fr]) -> Option<Polynomial> {
    let (a, b, c) = self.compute_witness_polys(witness);

    let ab = &a * &b;
    let p = &ab - &c;

    let (h, rem) = p.div_rem(&self.t);

    if rem.is_zero() {
        Some(h)    // valid witness → h(x) 존재
    } else {
        None       // invalid witness → 나누어떨어지지 않음
    }
}
```

### 완전한 수동 추적 ①: x·y=z 회로의 QAP

```
회로: x * y = z
witness: x=3, y=4, z=12

━━━ R1CS ━━━
변수: s = [1, 12, 3, 4]  (One, z(inst), x(wit), y(wit))
       idx: 0   1   2  3

제약 1개: x·y = z
  A[0] = [(2, 1)]   → 열 2에 1
  B[0] = [(3, 1)]   → 열 3에 1
  C[0] = [(1, 1)]   → 열 1에 1

행렬 (dense):
  A = [0, 0, 1, 0]
  B = [0, 0, 0, 1]
  C = [0, 1, 0, 0]

━━━ QAP 변환 ━━━
도메인: D = {1}  (m=1이므로)
t(x) = x - 1

열별 보간 (m=1이므로 각 다항식은 상수):
  a₀(x): A[0,0]=0 at ω=1 → a₀(x) = 0
  a₁(x): A[0,1]=0 at ω=1 → a₁(x) = 0
  a₂(x): A[0,2]=1 at ω=1 → a₂(x) = 1  (상수)
  a₃(x): A[0,3]=0 at ω=1 → a₃(x) = 0

  b₀(x) = 0,  b₁(x) = 0,  b₂(x) = 0,  b₃(x) = 1
  c₀(x) = 0,  c₁(x) = 1,  c₂(x) = 0,  c₃(x) = 0

━━━ Witness 결합 ━━━
  a(x) = s₀·a₀ + s₁·a₁ + s₂·a₂ + s₃·a₃
       = 1·0 + 12·0 + 3·1 + 4·0
       = 3  (상수)

  b(x) = 4  (상수)
  c(x) = 12 (상수)

━━━ h(x) 계산 ━━━
  p(x) = a(x)·b(x) - c(x) = 3·4 - 12 = 0

  h(x) = 0 / (x-1) = 0

  항등식: a(x)·b(x) - c(x) = 0 = 0·(x-1) = h(x)·t(x) ✓
```

> [!abstract] 가장 단순한 QAP
> 1개 제약 → degree 0 다항식들 (모두 상수)
> → a·b - c가 상수 0이면 자동으로 t(x)로 나누어떨어짐
> → h(x) = 0

```

#### 모든 15개 열 다항식의 완전한 계산

> [!abstract] 이 섹션의 목적
> 위에서 a₂(x), a₀(x), b₂(x) 세 개만 계산했다. 여기서는 Cubic 회로의
> **모든 15개 열 다항식** (A 5개 + B 5개 + C 5개)을 빠짐없이 계산한다.
> 이것이 QAP 변환의 "완전한 그림"이다.

```
복습: Dense 행렬 (3×5), 변수 순서: [One, y, x, t₁, t₂]

  A = [ 0,  0,  1,  0,  0 ]   ← 제약 ①: x·x = t₁
      [ 0,  0,  0,  1,  0 ]   ← 제약 ②: t₁·x = t₂
      [ 5,  0,  1,  0,  1 ]   ← 제약 ③: (t₂+x+5)·1 = y

  B = [ 0,  0,  1,  0,  0 ]   ← x
      [ 0,  0,  1,  0,  0 ]   ← x
      [ 1,  0,  0,  0,  0 ]   ← 1

  C = [ 0,  0,  0,  1,  0 ]   ← t₁
      [ 0,  0,  0,  0,  1 ]   ← t₂
      [ 0,  1,  0,  0,  0 ]   ← y

도메인: D = {1, 2, 3}

  L₀(x) = (x-2)(x-3)/2   = (1/2)x² - (5/2)x + 3
  L₁(x) = -(x-1)(x-3)    = -x² + 4x - 3

```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  2. 비영 다항식 9개, 각각 degree ≤ 2 (= m-1 = 3-1)
  3. b₀(x) = c₁(x) = a₄(x) — 우연의 일치가 아님!
     B의 One 열, C의 y 열, A의 t₂ 열이 모두 [0,0,1]이므로 같은 L₂(x)
  4. a₃(x) = c₄(x) — 마찬가지로 둘 다 [0,1,0]이므로 같은 L₁(x)

> [!tip] 희소성의 위력
> ```
> 15개 열 다항식 중:
>   전부 0인 열: 6개 (40%)  → 보간 완전 생략
>   L₀만: 2개 (13%)
>   L₁만: 2개 (13%)
>   L₂만: 3개 (20%)
>   혼합:  2개 (13%)        → a₂(x) = L₀+L₂, b₂(x) = L₀+L₁
>
> 대부분의 열 다항식은 Lagrange 기저 하나이거나 0이다!
> R1CS 행렬이 희소하므로 대부분의 열에는 비영 원소가 0~1개뿐.
> 이것이 QAP가 실제로는 보이는 것보다 훨씬 "가벼운" 이유이다.
> ```

### Witness 결합의 상세 수동 추적: 다항식 계수 레벨

> [!abstract] 이 섹션의 목적
> 위의 "도메인 점에서 검증" 섹션에서는 a(ωᵢ), b(ωᵢ), c(ωᵢ)를 행렬 값으로 직접 계산했다.
> 여기서는 **실제 다항식의 계수**를 구하여, 도메인 밖의 임의 점에서도 평가할 수 있는
> a(x), b(x), c(x)를 완전히 결정한다.

```
s = [1, 35, 3, 9, 27]  (One=1, y=35, x=3, t₁=9, t₂=27)

━━━ a(x) = Σⱼ sⱼ · aⱼ(x) ━━━

  = 1·a₀(x) + 35·a₁(x) + 3·a₂(x) + 9·a₃(x) + 27·a₄(x)
  = 1·[(5/2)x²-(15/2)x+5] + 35·0 + 3·[x²-4x+4] + 9·[-x²+4x-3] + 27·[(1/2)x²-(3/2)x+1]

  x² 계수: 5/2 + 3·1 + 9·(-1) + 27·(1/2) = 5/2 + 3 - 9 + 27/2
          = 5/2 + 27/2 + 3 - 9 = 32/2 - 6 = 16 - 6 = 10
  x¹ 계수: -15/2 + 3·(-4) + 9·4 + 27·(-3/2)
          = -15/2 - 12 + 36 - 81/2 = (-15-81)/2 + 24 = -96/2 + 24 = -48 + 24 = -24
  x⁰ 계수: 5 + 3·4 + 9·(-3) + 27·1 = 5 + 12 - 27 + 27 = 17

  a(x) = 10x² - 24x + 17

  검증:
    a(1) = 10 - 24 + 17 = 3  ✓ (= ⟨A[0], s⟩)
    a(2) = 40 - 48 + 17 = 9  ✓ (= ⟨A[1], s⟩)
    a(3) = 90 - 72 + 17 = 35 ✓ (= ⟨A[2], s⟩)


━━━ b(x) = Σⱼ sⱼ · bⱼ(x) ━━━

  = 1·b₀(x) + 35·0 + 3·b₂(x) + 9·0 + 27·0
  = 1·[(1/2)x²-(3/2)x+1] + 3·[-(1/2)x²+(3/2)x]

  x² 계수: 1/2 + 3·(-1/2) = 1/2 - 3/2 = -1
  x¹ 계수: -3/2 + 3·(3/2) = -3/2 + 9/2 = 3
  x⁰ 계수: 1 + 3·0 = 1

  b(x) = -x² + 3x + 1

  검증:
    b(1) = -1 + 3 + 1 = 3   ✓ (= ⟨B[0], s⟩)
    b(2) = -4 + 6 + 1 = 3   ✓ (= ⟨B[1], s⟩)
    b(3) = -9 + 9 + 1 = 1   ✓ (= ⟨B[2], s⟩)


━━━ c(x) = Σⱼ sⱼ · cⱼ(x) ━━━

  = 1·0 + 35·c₁(x) + 3·0 + 9·c₃(x) + 27·c₄(x)
  = 35·[(1/2)x²-(3/2)x+1] + 9·[(1/2)x²-(5/2)x+3] + 27·[-x²+4x-3]

  x² 계수: 35/2 + 9/2 + 27·(-1) = 44/2 - 27 = 22 - 27 = -5
  x¹ 계수: 35·(-3/2) + 9·(-5/2) + 27·4 = -105/2 - 45/2 + 108
          = -150/2 + 108 = -75 + 108 = 33
  x⁰ 계수: 35·1 + 9·3 + 27·(-3) = 35 + 27 - 81 = -19

  c(x) = -5x² + 33x - 19

  검증:
    c(1) = -5 + 33 - 19 = 9   ✓ (= ⟨C[0], s⟩)
    c(2) = -20 + 66 - 19 = 27 ✓ (= ⟨C[1], s⟩)
    c(3) = -45 + 99 - 19 = 35 ✓ (= ⟨C[2], s⟩)
```

```
━━━ Witness 결합 다항식 요약 ━━━

  a(x) = 10x² - 24x + 17    (degree 2)
  b(x) = -x² + 3x + 1       (degree 2)
  c(x) = -5x² + 33x - 19    (degree 2)

  ┌──────────────────────────────────────────────────────────────┐
  │  모든 다항식이 degree 2 (= m-1 = 3-1)                        │
  │                                                               │
  │  도메인 점에서의 값은 R1CS의 ⟨행, witness⟩와 정확히 일치:      │
  │                                                               │
  │  x=1: a=3,  b=3,  c=9    → a·b = 9 = c   ✓ (제약 ①)         │
  │  x=2: a=9,  b=3,  c=27   → a·b = 27 = c  ✓ (제약 ②)         │
  │  x=3: a=35, b=1,  c=35   → a·b = 35 = c  ✓ (제약 ③)         │
  │                                                               │
  │  도메인 밖 (예: x=5):                                          │
  │    a(5) = 250 - 120 + 17 = 147                                │
  │    b(5) = -25 + 15 + 1 = -9                                   │
  │    c(5) = -125 + 165 - 19 = 21                                │
  │    a·b = 147·(-9) = -1323 ≠ 21 = c                            │
  │    → 도메인 밖에서는 a·b ≠ c — 이것이 정상!                     │
  │    → p(5) = -1323 - 21 = -1344 ≠ 0                            │
  │    → 하지만 p(x)가 t(x)로 나누어떨어지기만 하면 OK               │
  └──────────────────────────────────────────────────────────────┘
```

### h(x) 계수의 완전한 수동 계산

> [!abstract] 이 섹션의 목적
> "h(x)가 존재한다"는 것만 확인하는 것이 아니라,
> **실제로 h(x)의 계수가 무엇인지** 직접 계산한다.
> 이것은 Groth16에서 증명자가 실제로 수행하는 연산의 핵심이다.

```
Step 1: p(x) = a(x)·b(x) - c(x) 계산

a(x) = 10x² - 24x + 17
b(x) = -x² + 3x + 1

a(x)·b(x) = (10x² - 24x + 17)(-x² + 3x + 1)

schoolbook 곱셈 (각 항의 곱을 차수별로 정리):

  10x² · (-x²) = -10x⁴
  10x² · 3x    = 30x³
  10x² · 1     = 10x²

  -24x · (-x²) = 24x³
  -24x · 3x    = -72x²
  -24x · 1     = -24x

  17 · (-x²)   = -17x²
  17 · 3x      = 51x
  17 · 1       = 17

차수별 정리:
  x⁴: -10
  x³: 30 + 24 = 54
  x²: 10 - 72 - 17 = -79
  x¹: -24 + 51 = 27
  x⁰: 17

a(x)·b(x) = -10x⁴ + 54x³ - 79x² + 27x + 17

p(x) = a(x)·b(x) - c(x)
     = (-10x⁴ + 54x³ - 79x² + 27x + 17) - (-5x² + 33x - 19)
     = -10x⁴ + 54x³ + (-79 + 5)x² + (27 - 33)x + (17 + 19)
     = -10x⁴ + 54x³ - 74x² - 6x + 36

검증 (도메인 점에서 0이어야 함):
  p(1) = -10 + 54 - 74 - 6 + 36 = 0 ✓
  p(2) = -160 + 432 - 296 - 12 + 36 = 0 ✓
  p(3) = -810 + 1458 - 666 - 18 + 36 = 0 ✓
```

```
Step 2: p(x) ÷ t(x) = h(x) 다항식 long division

p(x) = -10x⁴ + 54x³ - 74x² - 6x + 36
t(x) = x³ - 6x² + 11x - 6

━━━ Division Step 1 ━━━
  최고차 항: -10x⁴ / x³ = -10x

  -10x · t(x) = -10x · (x³ - 6x² + 11x - 6)
              = -10x⁴ + 60x³ - 110x² + 60x

  remainder = p(x) - (-10x · t(x))
            = (-10x⁴ + 54x³ - 74x² - 6x + 36) - (-10x⁴ + 60x³ - 110x² + 60x)
            = (54 - 60)x³ + (-74 + 110)x² + (-6 - 60)x + 36
            = -6x³ + 36x² - 66x + 36

━━━ Division Step 2 ━━━
  최고차 항: -6x³ / x³ = -6

  -6 · t(x) = -6 · (x³ - 6x² + 11x - 6)
            = -6x³ + 36x² - 66x + 36

  remainder = (-6x³ + 36x² - 66x + 36) - (-6x³ + 36x² - 66x + 36)
            = 0

━━━ 결과 ━━━
  h(x) = -10x + (-6) = -10x - 6
  remainder = 0 ✓ (나누어떨어짐! → QAP 만족)

  h(x)의 계수: coeffs = [-6, -10]
  degree(h) = 1 ≤ m - 2 = 1 ✓
```

```
Step 3: 항등식 검증 — 임의의 점 τ = 42에서

a(42)·b(42) - c(42) = h(42)·t(42)  를 확인

좌변:
  a(42) = 10·1764 - 24·42 + 17 = 17640 - 1008 + 17 = 16649
  b(42) = -1764 + 126 + 1 = -1637
  c(42) = -5·1764 + 33·42 - 19 = -8820 + 1386 - 19 = -7453

  a(42)·b(42) = 16649 · (-1637) = -27,254,213
  a(42)·b(42) - c(42) = -27,254,213 - (-7453) = -27,254,213 + 7453 = -27,246,760

우변:
  h(42) = -10·42 - 6 = -426
  t(42) = 42³ - 6·42² + 11·42 - 6 = 74088 - 10584 + 462 - 6 = 63960

  h(42)·t(42) = -426 · 63960 = -27,246,960

⚠️ 정수 산술로는 좌변 ≠ 우변이지만, Fr 체 위에서는 mod r 연산으로 일치한다!
(BN254 Fr에서의 계산은 오버플로가 mod r로 처리되므로,
 테스트 코드에서 assert_eq!(lhs, rhs)가 통과한다)

실제로 코드가 검증하는 것:
  assert_eq!(
      a.eval(Fr::from_u64(42)) * b.eval(Fr::from_u64(42)) - c.eval(Fr::from_u64(42)),
      h.eval(Fr::from_u64(42)) * qap.t.eval(Fr::from_u64(42))
  );
→ Fr 체 위에서 완벽히 일치 ✓
```

> [!tip] h(x)의 의미
> ```
> h(x) = -10x - 6
>
> 이 1차 다항식이 "3개 제약이 모두 만족된다"는 증거이다.
>
> Groth16에서 증명자는:
>   [H]₁ = h₀·[τ⁰]₁ + h₁·[τ¹]₁
>        = (-6)·[1]₁ + (-10)·[τ]₁
>
> 계수 h₀=-6, h₁=-10만 있으면 [H]₁을 계산할 수 있다.
> 이것이 h(x)의 "계수를 알아야 한다"는 의미이다.
>
> 단순히 h(τ)의 값 하나가 아니라,
> h(x)라는 다항식 전체 (= 모든 계수)를 알아야 증명을 생성할 수 있다.
> ```

### 완전한 수동 추적 ② (계속): 도메인 점에서의 검증

> [!note] 이 섹션의 위치
> 위에서 모든 열 다항식과 witness 결합 다항식의 계수를 구했다.
> 여기서는 도메인 점에서의 평가를 통해 R1CS 제약이 만족되는 것을 검증한다.
> (행렬 원소를 직접 사용하는 "빠른 방법"으로 같은 결과를 확인)

```
━━━ Witness 결합 (도메인 점에서 검증) ━━━

  s = [1, 35, 3, 9, 27]

  ω₁ = 1에서:
    a(1) = Σ sⱼ·aⱼ(1) = Σ sⱼ·A[0,j] = 0·1 + 0·35 + 1·3 + 0·9 + 0·27 = 3
    b(1) = Σ sⱼ·bⱼ(1) = Σ sⱼ·B[0,j] = 0 + 0 + 1·3 + 0 + 0 = 3
    c(1) = Σ sⱼ·cⱼ(1) = Σ sⱼ·C[0,j] = 0 + 0 + 0 + 1·9 + 0 = 9

    a(1)·b(1) = 3·3 = 9 = c(1) ✓  (제약 ①: x·x = t₁)

  ω₂ = 2에서:
    a(2) = 0 + 0 + 0 + 1·9 + 0 = 9     (A[1]: t₁)
    b(2) = 0 + 0 + 1·3 + 0 + 0 = 3     (B[1]: x)
    c(2) = 0 + 0 + 0 + 0 + 1·27 = 27   (C[1]: t₂)

    a(2)·b(2) = 9·3 = 27 = c(2) ✓  (제약 ②: t₁·x = t₂)

  ω₃ = 3에서:
    a(3) = 5·1 + 0 + 1·3 + 0 + 1·27 = 35  (A[2]: t₂+x+5)
    b(3) = 1·1 + 0 + 0 + 0 + 0 = 1        (B[2]: 1)
    c(3) = 0 + 1·35 + 0 + 0 + 0 = 35      (C[2]: y)

    a(3)·b(3) = 35·1 = 35 = c(3) ✓  (제약 ③: (t₂+x+5)·1 = y)

━━━ h(x) 계산 ━━━
  p(x) = a(x)·b(x) - c(x)

  degree(a) ≤ 2, degree(b) ≤ 2 → degree(a·b) ≤ 4
  degree(c) ≤ 2
  degree(p) ≤ 4
  degree(t) = 3

  → degree(h) ≤ 4 - 3 = 1  (최대 1차)

  p(1) = a(1)·b(1) - c(1) = 9 - 9 = 0 ✓
  p(2) = a(2)·b(2) - c(2) = 27 - 27 = 0 ✓
  p(3) = a(3)·b(3) - c(3) = 35 - 35 = 0 ✓

  → p(x)는 x=1,2,3에서 0 → t(x)=(x-1)(x-2)(x-3)로 나누어떨어짐

━━━ 항등식 검증 (τ = 42) ━━━
  a(42)·b(42) - c(42) = h(42)·t(42)

  테스트 코드에서 이것을 검증:
    let tau = Fr::from_u64(42);
    let lhs = a.eval(tau) * b.eval(tau) - c.eval(tau);
    let rhs = h.eval(tau) * qap.t.eval(tau);
    assert_eq!(lhs, rhs);  // ✓
```

### 잘못된 witness로 QAP가 실패하는 과정 — 수동 추적

> [!important] 이 섹션의 목적
> 올바른 witness에서 QAP가 작동하는 과정은 위에서 보았다.
> 이제 **잘못된** witness를 넣었을 때 정확히 어디서, 어떻게 실패하는지를
> 한 단계씩 추적한다. 이것이 QAP의 **건전성(soundness)**을 직관적으로 이해하는 열쇠이다.

#### 설정: x=3이지만 y=999 (올바른 값은 y=35)

```
회로: f(x) = x³ + x + 5 = y

올바른 witness: x=3 → t₁=9, t₂=27, y=35
잘못된 witness: x=3 → t₁=9, t₂=27, y=999  (y를 조작!)

s_bad = [1, 999, 3, 9, 27]
         ↑    ↑
         One  y(잘못됨!)

R1CS 제약 3개:
  ① x·x = t₁         → A[0]=[(2,1)], B[0]=[(2,1)], C[0]=[(3,1)]
  ② t₁·x = t₂        → A[1]=[(3,1)], B[1]=[(2,1)], C[1]=[(4,1)]
  ③ (t₂+x+5)·1 = y   → A[2]=[(4,1),(2,1),(0,5)], B[2]=[(0,1)], C[2]=[(1,1)]
```

#### Step 1: 도메인 점에서 a(ωᵢ), b(ωᵢ), c(ωᵢ) 계산

```
QAP의 열 다항식 aⱼ(x), bⱼ(x), cⱼ(x)는 R1CS 행렬에만 의존한다.
→ witness가 바뀌어도 열 다항식은 동일!

Witness 결합:
  a(x) = Σⱼ s_bad[j] · aⱼ(x)
  b(x) = Σⱼ s_bad[j] · bⱼ(x)
  c(x) = Σⱼ s_bad[j] · cⱼ(x)

도메인 점에서 평가 (핵심: aⱼ(ωᵢ) = A[i,j]):

━━━ ω₁ = 1 (제약 ①: x·x = t₁) ━━━

  a(1) = Σ s_bad[j]·A[0,j]
       = 1·0 + 999·0 + 3·1 + 9·0 + 27·0
       = 3

  b(1) = Σ s_bad[j]·B[0,j]
       = 1·0 + 999·0 + 3·1 + 9·0 + 27·0
       = 3

  c(1) = Σ s_bad[j]·C[0,j]
       = 1·0 + 999·0 + 3·0 + 9·1 + 27·0
       = 9

  확인: a(1)·b(1) = 3·3 = 9 = c(1) ✓
  → 제약 ①은 y와 무관하므로 여전히 만족!


━━━ ω₂ = 2 (제약 ②: t₁·x = t₂) ━━━

  a(2) = Σ s_bad[j]·A[1,j]
       = 1·0 + 999·0 + 3·0 + 9·1 + 27·0
       = 9

  b(2) = Σ s_bad[j]·B[1,j]
       = 1·0 + 999·0 + 3·1 + 9·0 + 27·0
       = 3

  c(2) = Σ s_bad[j]·C[1,j]
       = 1·0 + 999·0 + 3·0 + 9·0 + 27·1
       = 27

  확인: a(2)·b(2) = 9·3 = 27 = c(2) ✓
  → 제약 ②도 y와 무관하므로 여전히 만족!


━━━ ω₃ = 3 (제약 ③: (t₂+x+5)·1 = y) ━━━ ← 여기서 실패!

  a(3) = Σ s_bad[j]·A[2,j]
       = 1·5 + 999·0 + 3·1 + 9·0 + 27·1
       = 5 + 3 + 27
       = 35

  b(3) = Σ s_bad[j]·B[2,j]
       = 1·1 + 999·0 + 3·0 + 9·0 + 27·0
       = 1

  c(3) = Σ s_bad[j]·C[2,j]
       = 1·0 + 999·1 + 3·0 + 9·0 + 27·0
       = 999          ← 잘못된 y가 여기에 반영!

  확인: a(3)·b(3) = 35·1 = 35 ≠ 999 = c(3)  ✗✗✗
  → 제약 ③ 불만족!
  → 차이: 35 - 999 = -964
```

#### Step 2: p(x)가 ω₃에서 0이 아님을 확인

```
p(x) = a(x)·b(x) - c(x)

도메인 점에서 p의 값:
  p(1) = a(1)·b(1) - c(1) = 9 - 9     = 0    ✓ (제약 ① 만족)
  p(2) = a(2)·b(2) - c(2) = 27 - 27   = 0    ✓ (제약 ② 만족)
  p(3) = a(3)·b(3) - c(3) = 35 - 999  = -964 ✗ (제약 ③ 불만족!)

  ┌───────────────────────────────────────────────────┐
  │  p(x)의 값:                                        │
  │                                                    │
  │    p(1) = 0     ← ω₁은 p(x)의 근                   │
  │    p(2) = 0     ← ω₂도 p(x)의 근                   │
  │    p(3) = -964  ← ω₃는 p(x)의 근이 아님! ← 핵심!    │
  │                                                    │
  │  소거 다항식: t(x) = (x-1)(x-2)(x-3)               │
  │    t(x)의 근: {1, 2, 3}                             │
  │                                                    │
  │  p(x)의 근: {1, 2}만 — 3은 근이 아님!               │
  │  → (x-3)은 p(x)의 인수가 아님                       │
  │  → t(x) = (x-1)(x-2)(x-3)은 p(x)를 나눌 수 없음   │
  └───────────────────────────────────────────────────┘
```

#### Step 3: p(x) / t(x)에서 나머지가 발생

```
p(x)의 구조 분석:

  p(x)는 ω₁=1, ω₂=2에서만 0이므로:
    p(x) = (x - 1)(x - 2) · q(x)   (어떤 다항식 q(x)에 대해)

  그런데 t(x) = (x-1)(x-2)(x-3)이므로:
    p(x) / t(x) = (x-1)(x-2)·q(x) / ((x-1)(x-2)(x-3))
                = q(x) / (x - 3)

  q(3) = p(3) / ((3-1)(3-2)) = -964 / (2·1) = -482
  → q(3) ≠ 0 → (x-3)은 q(x)의 인수가 아님
  → q(x) / (x-3)는 나누어떨어지지 않음!

다항식 long division 수행:
  p(x) ÷ t(x) = h(x) ... remainder

  결과: remainder ≠ 0

  ┌──────────────────────────────────────────────────┐
  │                                                   │
  │   p(x) = h(x) · t(x) + remainder(x)              │
  │                         ─────────────             │
  │                         이것이 0이 아님!            │
  │                                                   │
  │   remainder ≠ 0                                   │
  │   → h(x)가 다항식으로 존재하지 않음                  │
  │   → QAP 불만족                                     │
  │   → 증명 생성 불가!                                 │
  │                                                   │
  └──────────────────────────────────────────────────┘
```

#### Step 4: 코드에서의 검출

```rust
// qap.rs의 compute_h 함수
pub fn compute_h(&self, witness: &[Fr]) -> Option<Polynomial> {
    let (a, b, c) = self.compute_witness_polys(witness);
    let ab = &a * &b;
    let p = &ab - &c;         // p(x) = a(x)·b(x) - c(x)

    let (h, rem) = p.div_rem(&self.t);  // p(x) ÷ t(x)

    if rem.is_zero() {
        Some(h)     // valid → h(x) 존재
    } else {
        None        // invalid → remainder ≠ 0!  ← 여기로 진입!
    }
}

// 테스트에서의 사용:
// s_bad = [1, 999, 3, 9, 27]
// let result = qap.compute_h(&s_bad);
// assert!(result.is_none());  // ✓ QAP 거부!
```

```

> [!tip] 핵심 교훈
> 잘못된 witness는 **단 하나의 제약이라도** 불만족하면 QAP 전체가 실패한다.
>
> 이유: 하나의 제약 ωᵢ에서 p(ωᵢ) ≠ 0이면,
> (x - ωᵢ)가 p(x)의 인수가 아니므로,
> t(x) = ∏(x - ωⱼ)는 p(x)를 나눌 수 없다.
>
> **"하나의 거짓이 전체를 무너뜨린다"** --- 이것이 QAP 건전성의 핵심이다.
> 다항식의 인수 구조가 이 "all-or-nothing" 성질을 보장한다.

### 차수 분석 (Degree Analysis)

```
m = 제약 수, n = 변수 수

다항식 차수:
  aⱼ(x): degree ≤ m-1  (m개의 점을 보간)
  bⱼ(x): degree ≤ m-1
  cⱼ(x): degree ≤ m-1

  a(x) = Σ sⱼ·aⱼ(x): degree ≤ m-1
  b(x) = Σ sⱼ·bⱼ(x): degree ≤ m-1
  c(x) = Σ sⱼ·cⱼ(x): degree ≤ m-1

  a(x)·b(x): degree ≤ 2(m-1)
  t(x): degree = m

  p(x) = a(x)·b(x) - c(x): degree ≤ 2(m-1)
  h(x) = p(x)/t(x): degree ≤ 2(m-1) - m = m - 2

Groth16에서의 의미:
  CRS(Common Reference String)에 필요한 powers of τ:
    [τ⁰]₁, [τ¹]₁, ..., [τ^(m-1)]₁   ← a, b, c 평가용
    [τ⁰]₁, [τ¹]₁, ..., [τ^(m-2)]₁   ← h 평가용

  → CRS 크기 ∝ m (= 제약 수)
  → 증명 크기: 일정 (G1 2개 + G2 1개 = 128 bytes)
  → 검증 시간: 일정 (pairing 3~4회)

구체적 수치 (3차 다항식 회로):
  m = 3 → a,b,c degree ≤ 2, h degree ≤ 1, t degree = 3
```

> [!important] 차수와 효율성
> ```
>             우리 교육용      Poseidon 회로      대규모 회로
> 제약 수 m:  3               634                100,000
> a,b,c deg:  ≤ 2             ≤ 633              ≤ 99,999
> h deg:      ≤ 1             ≤ 632              ≤ 99,998
> t deg:      3               634                100,000
> CRS 크기:   ~3              ~634               ~100,000 (G1 points)
> 증명 크기:  128B            128B               128B (!!)
> 검증 시간:  O(1)            O(1)               O(1)
>
> 제약 수가 아무리 커도 증명 크기와 검증 시간은 일정!
> 이것이 SNARK의 "Succinct"(간결한)의 의미.
> ```

### QAP 구조체

```rust
/// QAP (Quadratic Arithmetic Program)
pub struct QAP {
    pub a_polys: Vec<Polynomial>,  // aⱼ(x): A 열별 다항식 (n개)
    pub b_polys: Vec<Polynomial>,  // bⱼ(x): B 열별 다항식 (n개)
    pub c_polys: Vec<Polynomial>,  // cⱼ(x): C 열별 다항식 (n개)
    pub t: Polynomial,             // t(x): 소거 다항식
    pub domain: Vec<Fr>,           // {ω₁, ..., ωₘ}
    pub num_instance: usize,
    pub num_variables: usize,
}
```

```
QAP의 메모리 사용:
  a_polys: n × m 계수 (각 다항식 degree ≤ m-1)
  b_polys: n × m
  c_polys: n × m
  t: m+1 계수

  총: ~3nm 개의 Fr 원소 = 3nm × 32 bytes

  3차 다항식 회로: 3 × 5 × 3 × 32 = 1,440 bytes
  Poseidon 회로: 3 × 638 × 634 × 32 ≈ 38 MB
  → 대규모 회로에서는 이 행렬 기반 방식이 비효율적
  → 프로덕션: FFT로 필요할 때만 평가
```

### QAP 보안 증명: 왜 h(x)의 존재가 건전성을 보장하는가

> [!important] 핵심 정리
> **QAP 건전성 (Soundness)**: h(x)가 다항식으로 존재한다 ⟺ R1CS가 만족된다.
> 이 동치 관계가 ZK-SNARK의 수학적 기반이다.
> 한쪽만 성립해서는 안 된다 --- 양방향 모두 증명해야 한다.

#### 정방향 증명: h(x) 존재 → R1CS 만족

```
정리: a(x)·b(x) - c(x) = h(x)·t(x)인 다항식 h(x)가 존재하면,
      모든 i에 대해 ⟨A[i],s⟩·⟨B[i],s⟩ = ⟨C[i],s⟩ 이다.

증명:
  가정: a(x)·b(x) - c(x) = h(x)·t(x)  (다항식 항등식)

  임의의 도메인 점 ωᵢ에서 양변을 평가:

    좌변: a(ωᵢ)·b(ωᵢ) - c(ωᵢ)

    우변: h(ωᵢ)·t(ωᵢ)
        = h(ωᵢ)·0         ← t(ωᵢ) = 0 (소거 다항식의 정의!)
        = 0

  따라서:
    a(ωᵢ)·b(ωᵢ) - c(ωᵢ) = 0
    a(ωᵢ)·b(ωᵢ) = c(ωᵢ)

  그런데 witness 결합에 의해:
    a(ωᵢ) = Σⱼ sⱼ·aⱼ(ωᵢ) = Σⱼ sⱼ·A[i,j] = ⟨A[i], s⟩
    b(ωᵢ) = ⟨B[i], s⟩
    c(ωᵢ) = ⟨C[i], s⟩

  따라서:
    ⟨A[i],s⟩·⟨B[i],s⟩ = ⟨C[i],s⟩   ∀i ∈ {1,...,m}

  이것은 정확히 R1CS의 정의이다. □
```

```
직관적 이해:

  t(x) = (x - ω₁)(x - ω₂)···(x - ωₘ)
           │         │              │
           ▼         ▼              ▼
         t(ω₁)=0  t(ω₂)=0  ···  t(ωₘ)=0

  h(x)·t(x)는 모든 ωᵢ에서 0이다 (t가 0이므로)
  → a·b - c도 모든 ωᵢ에서 0이어야 한다
  → 모든 R1CS 제약이 만족된다

  핵심: t(x)의 "소거" 성질이 증명의 열쇠
  소거 다항식은 이름 그대로, 도메인 점에서 우변을 "소거"한다.
```

#### 역방향 증명: R1CS 만족 → h(x) 존재

```
정리: 모든 i에 대해 ⟨A[i],s⟩·⟨B[i],s⟩ = ⟨C[i],s⟩이면,
      a(x)·b(x) - c(x) = h(x)·t(x)인 다항식 h(x)가 존재한다.

증명:
  p(x) = a(x)·b(x) - c(x) 로 정의

  가정에 의해 모든 i에 대해:
    p(ωᵢ) = a(ωᵢ)·b(ωᵢ) - c(ωᵢ)
           = ⟨A[i],s⟩·⟨B[i],s⟩ - ⟨C[i],s⟩
           = 0

  따라서 p(x)는 ω₁, ω₂, ..., ωₘ 을 근으로 가진다.

  대수학의 기본 정리:
    다항식 p(x)가 x = r에서 0이면, (x - r)이 p(x)의 인수이다.

  p(x)는 m개의 서로 다른 근을 가지므로:
    (x - ω₁), (x - ω₂), ..., (x - ωₘ) 이 모두 p(x)의 인수

  이 인수들은 서로 소 (ωᵢ가 모두 다르므로):
    → ∏ᵢ (x - ωᵢ) = t(x) 가 p(x)의 인수

  따라서:
    p(x) = h(x)·t(x)  (어떤 다항식 h(x)에 대해)
    → h(x) = p(x) / t(x)  (나머지 없이 나누어떨어짐) □
```

#### 양방향 동치의 의미

```
          h(x) 존재 (다항식으로 나누어떨어짐)
               ⟺
          R1CS 만족 (모든 제약 등식 성립)

이것이 의미하는 바:

  ┌─────────────────────────────────────────────────┐
  │ 검증자의 관점:                                    │
  │                                                   │
  │ "h(x)가 존재하는지 확인하는 것"                     │
  │  = "모든 R1CS 제약이 만족되는지 확인하는 것"         │
  │                                                   │
  │ m개의 등식을 일일이 확인하는 대신,                   │
  │ h(x)의 존재 여부만 확인하면 된다!                   │
  │                                                   │
  │ Schwartz-Zippel로 한 점에서 검증:                  │
  │   a(τ)·b(τ) - c(τ) = h(τ)·t(τ)?                  │
  │   → "예" → 모든 제약 만족 (확률 ≥ 1 - m/|Fr|)      │
  │   → "아니오" → 어떤 제약이 불만족 (확실)             │
  └─────────────────────────────────────────────────┘
```

#### Groth16에서의 적용: 비밀 τ와 건전성

```
문제: 순진한 프로토콜에서의 치팅 가능성

  순진한 프로토콜:
    ① 검증자가 공개 점 τ를 선택
    ② 증명자가 a(τ), b(τ), c(τ), h(τ) 계산하여 전송
    ③ 검증자가 a(τ)·b(τ) - c(τ) = h(τ)·t(τ) 확인

  치팅:
    증명자가 잘못된 witness를 사용 → p(x) = a(x)·b(x) - c(x) ≠ h(x)·t(x)
    하지만! τ를 알고 있으므로:
      → 가짜 h_fake = (a(τ)·b(τ) - c(τ)) / t(τ) 계산 가능
      → h_fake는 "숫자"이지 "다항식의 τ에서의 평가"가 아님
      → 하지만 검증자는 구별 불가!

  ┌─────────────────────────────────────────────────────────┐
  │ Groth16의 해결: τ를 비밀로 유지                           │
  │                                                          │
  │  Trusted Setup:                                          │
  │    τ 선택 후, [τⁱ]₁ = τⁱ·G₁ 만 공개                      │
  │    τ 자체는 폐기 (toxic waste)                            │
  │                                                          │
  │  증명자:                                                  │
  │    [h(τ)]₁ = Σ hᵢ·[τⁱ]₁  (다항식의 계수로 계산)          │
  │    → h(x)의 "모든 계수"를 알아야 계산 가능                 │
  │    → 한 점의 값만으로는 [h(τ)]₁ 생성 불가                  │
  │       (이산 로그 문제의 어려움)                             │
  │                                                          │
  │  검증자:                                                  │
  │    e([A]₁, [B]₂) = e([C]₁ + [H]₁·[t(τ)]₂, [1]₂)        │
  │    → pairing으로 "암호화된 상태에서" 항등식 검증            │
  │    → τ를 모르고도 검증 가능!                               │
  │                                                          │
  │  건전성:                                                  │
  │    증명자가 유효한 [h(τ)]₁을 생성하려면                    │
  │    실제 다항식 h(x)의 계수를 알아야 함                     │
  │    → h(x)가 존재해야 함                                   │
  │    → R1CS가 만족되어야 함                                 │
  └─────────────────────────────────────────────────────────┘
```

> [!abstract] 보안 체인 요약
> ```
> 건전성 체인:
>
>   유효한 증명 π 생성 가능
>     → [h(τ)]₁ 계산 가능
>     → h(x) 다항식의 계수를 알아야 함  (이산 로그 가정)
>     → h(x)가 다항식으로 존재해야 함
>     → p(x) = a(x)·b(x) - c(x)가 t(x)로 나누어떨어짐
>     → p(ωᵢ) = 0 for all i
>     → R1CS가 만족됨
>     → witness가 유효함
>
>   어느 한 단계라도 깨지면 전체 체인이 붕괴한다.
>   이것이 ZK-SNARK의 건전성 (soundness) 보장이다.
> ```

### 열(column) 보간 vs 행(row) 보간: 왜 열을 보간하는가?

> [!important] 이 섹션의 핵심 질문
> R1CS 행렬은 m행 n열이다 (m=제약 수, n=변수 수).
> "열별로" 보간하는 이유가 무엇인가? "행별로" 보간하면 안 되는가?

```
행 vs 열: 직관

  행 = 하나의 제약에 관여하는 모든 변수의 가중치
    A[i] = [0, 0, 1, 0, 0]  "제약 i에서 변수 x의 계수가 1"

  열 = 하나의 변수가 모든 제약에서 갖는 가중치
    A[*,j] = [1, 0, 1]  "변수 x가 제약 0에서 1, 제약 1에서 0, 제약 2에서 1"

QAP에서 열을 보간하는 이유:

  Witness 결합이 "변수별 가중합"이기 때문이다:
    a(x) = Σⱼ sⱼ · aⱼ(x)
            └─ 변수 j의 witness 값

  aⱼ(x)는 "변수 j가 각 제약에서 갖는 가중치"를 인코딩한 다항식.
  이것이 바로 행렬의 j번째 열이다.

  만약 행을 보간했다면?
    행 다항식 ā_i(x)는 "제약 i에 관여하는 변수별 가중치"를 인코딩
    → witness와의 결합이 "제약별"이 됨
    → 모든 제약을 하나의 다항식으로 합칠 수 없음!
```

```
구체적 예: Cubic 회로에서의 차이

열 보간 (우리가 하는 방식):
  a₂(x): 변수 x의 A 가중치를 제약별로 인코딩
    a₂(1) = A[0,2] = 1  (제약 0에서 x의 A 가중치)
    a₂(2) = A[1,2] = 0  (제약 1에서 x의 A 가중치)
    a₂(3) = A[2,2] = 1  (제약 2에서 x의 A 가중치)

  → a(x) = Σⱼ sⱼ · aⱼ(x)
    → a(ωᵢ) = Σⱼ sⱼ · A[i,j] = ⟨A[i], s⟩ ← R1CS의 i번째 내적!

행 보간 (하지 않는 방식):
  ā₀(x): 제약 0의 A 행 [0,0,1,0,0]을 변수 인덱스별로 인코딩
    ā₀(0) = A[0,0] = 0
    ā₀(1) = A[0,1] = 0
    ā₀(2) = A[0,2] = 1
    ...

  → 이것을 witness와 결합하려면?
    a₀ = Σⱼ sⱼ · ā₀(j) = ⟨A[0], s⟩  ← 하나의 스칼라!
    a₁ = Σⱼ sⱼ · ā₁(j)
    ...
    → m개의 스칼라가 나옴. 다항식이 아님!

핵심:
  ┌─────────────────────────────────────────────────────────┐
  │                                                          │
  │  열 보간: aⱼ(x)는 "변수 j의 제약별 역할"                  │
  │    → witness 결합 후 a(x)는 x에 대한 다항식               │
  │    → x = ωᵢ에서 평가하면 제약 i의 내적이 나옴              │
  │                                                          │
  │  행 보간: 의미 있는 다항식 구조를 만들 수 없음               │
  │                                                          │
  │  결론: 열 보간이 유일하게 의미 있는 선택                     │
  │                                                          │
  └─────────────────────────────────────────────────────────┘
```

```
시각화: 열 보간의 데이터 흐름

  R1CS 행렬 A (3×5):
       col 0  col 1  col 2  col 3  col 4
  row 0: 0     0      1      0      0     ← 제약 ①
  row 1: 0     0      0      1      0     ← 제약 ②
  row 2: 5     0      1      0      1     ← 제약 ③

  열 보간 (각 열을 세로로 추출):
         │              │              │
    col 0│ [0,0,5]  col 2│ [1,0,1]  col 3│ [0,1,0]
         │              │              │
         ↓              ↓              ↓
    Lagrange 보간   Lagrange 보간   Lagrange 보간
         │              │              │
         ↓              ↓              ↓
    a₀(x)         a₂(x)          a₃(x)

  도메인 점과의 대응:
    ω₁=1 ← row 0
    ω₂=2 ← row 1
    ω₃=3 ← row 2

  따라서: aⱼ(ωᵢ) = A[i, j]  ← 행렬의 (i,j) 원소!
```

> [!note] 수학적으로 더 정확한 표현
> ```
> 열 보간은 다음을 구성한다:
>
>   aⱼ: D → Fr  (도메인에서 Fr로의 함수)
>   aⱼ(ωᵢ) = Aᵢⱼ
>
> 이것의 유일한 degree < m 다항식 확장을 구하는 것이 Lagrange 보간.
>
> witness 결합:
>   a(x) = Σⱼ sⱼ · aⱼ(x)
>
> 도메인 점 ωᵢ에서:
>   a(ωᵢ) = Σⱼ sⱼ · Aᵢⱼ = (A · s)ᵢ  ← 행렬-벡터 곱의 i번째 성분!
>
> 즉:
>   "열 다항식의 witness 결합을 도메인에서 평가"
>   = "행렬-벡터 곱"
>   = "R1CS의 선형결합 평가"
>
> 이 동치가 QAP의 수학적 핵심이다.
> ```

### columns_to_polynomials 함수의 상세 코드 워크스루

```rust
fn columns_to_polynomials(

```

```
입력 형식 이해:

  R1CS의 to_matrices()가 반환하는 희소 행렬:
    matrix[i] = [(col₁, val₁), (col₂, val₂), ...]
    → i번째 제약(행)에서 비영인 (열 인덱스, 값) 쌍들

  예: Cubic 회로의 A 행렬
    matrix[0] = [(2, 1)]           → A[0,2] = 1 (제약 ①에서 x의 계수)
    matrix[1] = [(3, 1)]           → A[1,3] = 1 (제약 ②에서 t₁의 계수)
    matrix[2] = [(4,1), (2,1), (0,5)]  → A[2,4]=1, A[2,2]=1, A[2,0]=5

  이것을 열별 다항식으로 변환해야 한다.
  행별 희소 표현 → 열별 정보 추출이 핵심 과제.
```

```rust
    let m = matrix.len();  // 제약 수 (= 행 수)

    (0..num_vars).map(|j| {   // 각 변수(열) j에 대해
        // 열 j의 모든 값 수집: (ωᵢ, A[i,j]) 쌍
        let mut all_zero = true;
        let points: Vec<(Fr, Fr)> = (0..m).map(|i| {
            // matrix[i]에서 열 j의 값 찾기 (없으면 0)
            let val = matrix[i].iter()
                .find(|&&(col, _)| col == j)  // 행 i에서 열 j 찾기
                .map(|&(_, v)| v)             // 찾으면 값 추출
                .unwrap_or(Fr::ZERO);         // 없으면 0 (희소!)

            if !val.is_zero() { all_zero = false; }
            (domain[i], val)  // (ωᵢ, A[i,j])
        }).collect();
```

```
열 j의 값 수집 과정 (Cubic A 행렬, j=2):

  i=0: matrix[0] = [(2,1)]
       → col=2 찾음! val = 1
       → point: (ω₁=1, 1)

  i=1: matrix[1] = [(3,1)]
       → col=2 없음. val = 0
       → point: (ω₂=2, 0)

  i=2: matrix[2] = [(4,1),(2,1),(0,5)]
       → col=2 찾음! val = 1
       → point: (ω₃=3, 1)

  결과: points = [(1, 1), (2, 0), (3, 1)]
  all_zero = false

  → Lagrange 보간 호출: a₂(x) = x² - 4x + 4
```

```
열 j의 값 수집 (j=1, y의 A 열):

  i=0: matrix[0] = [(2,1)]      → col=1 없음. val = 0
  i=1: matrix[1] = [(3,1)]      → col=1 없음. val = 0
  i=2: matrix[2] = [(4,1),(2,1),(0,5)]  → col=1 없음. val = 0

  결과: points = [(1, 0), (2, 0), (3, 0)]
  all_zero = true

    Lagrange 보간을 호출해도 결과는 0 다항식
    하지만 O(m²) 연산이 낭비됨
    → n개 변수 중 대부분이 0열이므로 절약 효과 큼
```

```rust

            Polynomial::zero()  // 최적화: 전부 0이면 보간 생략
        } else {
            Polynomial::lagrange_interpolate(&points)
        }
    }).collect()  // n개의 열 다항식 벡터 반환
}

> [!tip] 희소 행렬에서 열 접근의 비용
> ```
> 우리 구현의 비효율:
>   행별 희소 표현 → 열 j의 값을 찾으려면 모든 행을 선형 탐색

>   전체: O(n × m × nnz_per_row)
>
> 프로덕션의 최적화:
>   CSC (Compressed Sparse Column) 형식으로 변환 후 열 접근 O(1)
>   또는 전치 행렬을 미리 구축
>
> 우리 회로 규모 (m=3, n=5)에서는 이 비효율이 전혀 문제되지 않는다.
> ```

### 연산자 오버로딩의 설계: 왜 &Polynomial인가

```rust
// 참조(&)로 구현된 연산자들:
impl std::ops::Add for &Polynomial { ... }
impl std::ops::Sub for &Polynomial { ... }
impl std::ops::Mul for &Polynomial { ... }
```

```
왜 &Polynomial인가?

  Polynomial은 Vec<Fr>을 포함 → 소유권 이동 시 원본 사용 불가

  예: h(x)·t(x) 항등식 검증
    let ab = &a * &b;        // a, b 모두 이후에도 사용 가능
    let p = &ab - &c;        // ab도 c도 이후 사용 가능

  만약 Polynomial (소유권 이동)이었다면:
    let ab = a * b;           // a, b 소유권 이동 → 더 이상 사용 불가!
    let p = ab - c;           // ab도 이동
    // a.eval(tau)  ← 컴파일 에러! a가 이미 이동됨

  따라서 &Polynomial이 올바른 선택:
    ① 원본 데이터를 보존하면서 연산 가능
    ② 같은 다항식을 여러 연산에 재사용 가능
    ③ clone() 불필요 → 불필요한 메모리 할당 방지
```

```
다항식 곱셈의 상세 워크스루:

impl Mul for &Polynomial:
    fn mul(self, rhs: &Polynomial) -> Polynomial {
        // 결과 차수 = deg(self) + deg(rhs)
        // → 결과 계수 수 = self.len + rhs.len - 1
        let len = self.coeffs.len() + rhs.coeffs.len() - 1;
        let mut coeffs = vec![Fr::ZERO; len];

        // Schoolbook 곱셈: a(x)·b(x)의 x^(i+j) 계수 = Σ aᵢ·bⱼ
        for (i, &a) in self.coeffs.iter().enumerate() {
            for (j, &b) in rhs.coeffs.iter().enumerate() {
                coeffs[i + j] = coeffs[i + j] + a * b;
                //     ↑ i+j차 계수에 aᵢ·bⱼ를 누적
            }
        }
        Polynomial::from_coeffs(coeffs)  // trailing zeros 제거
    }

수동 추적: (x + 1)(x + 2) = x² + 3x + 2

  self.coeffs  = [1, 1]   (= 1 + x)
  rhs.coeffs   = [2, 1]   (= 2 + x)
  len = 2 + 2 - 1 = 3
  coeffs = [0, 0, 0]

  i=0, j=0: coeffs[0] += 1·2 = 2    → [2, 0, 0]
  i=0, j=1: coeffs[1] += 1·1 = 1    → [2, 1, 0]
  i=1, j=0: coeffs[1] += 1·2 = 2+1=3 → [2, 3, 0]
  i=1, j=1: coeffs[2] += 1·1 = 1    → [2, 3, 1]

  결과: 2 + 3x + x² ✓
```

### 다항식 나눗셈 (div_rem) 코드의 상세 워크스루

> [!abstract] 이 섹션의 목적
> div_rem은 QAP의 핵심 연산이다: h(x) = p(x)/t(x).
> 코드의 각 줄이 수학적으로 무엇을 하는지 상세히 분석한다.

```rust
pub fn div_rem(&self, divisor: &Self) -> (Self, Self) {
    // 전제 조건 검사
    if divisor.is_zero() {
        panic!("division by zero polynomial");
    }
    if self.is_zero() || self.degree() < divisor.degree() {
        return (Self::zero(), self.clone());
        // degree(dividend) < degree(divisor) → 몫=0, 나머지=dividend
    }
```

```
기본 아이디어: 수동 long division을 코드로 변환

  수동:
    1. 피제수의 최고차 항 / 제수의 최고차 항 = 몫의 한 항
    2. 제수 × 몫의 한 항을 빼서 차수 줄임
    3. 차수가 제수보다 작아질 때까지 반복

  코드에서의 핵심 관찰:
    배열 인덱스를 사용하므로 "최고차 항"이 "배열의 끝"
    역순(rev) 반복으로 높은 차수부터 처리
```

```rust
    let mut remainder = self.coeffs.clone();  // 피제수의 복사본 (원본 보존)
    let divisor_lead_inv = divisor.coeffs.last().unwrap().inv()
        .expect("leading coefficient must be non-zero");
    // ↑ 제수의 최고차 계수의 역원 (monic이 아닌 경우 필요)
    // 예: t(x) = x³ - 6x² + 11x - 6 → leading coeff = 1 → inv = 1

    let div_deg = divisor.degree();     // 제수의 차수
    let quot_deg = self.degree() - div_deg;  // 몫의 차수
```

```rust
    let mut quotient = vec![Fr::ZERO; quot_deg + 1];

    for i in (0..=quot_deg).rev() {
        // i: 몫의 i차 계수를 결정
        // remainder[i + div_deg]: 현재 나머지의 (i+div_deg)차 계수
        let coeff = remainder[i + div_deg] * divisor_lead_inv;
        quotient[i] = coeff;

        // remainder -= coeff · x^i · divisor
        for j in 0..=div_deg {
            remainder[i + j] = remainder[i + j] - coeff * divisor.coeffs[j];
        }
        // → remainder[i + div_deg]가 정확히 0이 됨 (설계상)
    }
```

```
수동 추적: p(x) / t(x), Cubic 회로

p(x) = -10x⁴ + 54x³ - 74x² - 6x + 36
t(x) = x³ - 6x² + 11x - 6

div_deg = 3, quot_deg = 4 - 3 = 1
divisor_lead_inv = 1⁻¹ = 1

remainder = [36, -6, -74, 54, -10]  (인덱스 = 차수)

━━━ i=1 (몫의 1차 계수) ━━━
  coeff = remainder[1+3] * 1 = remainder[4] * 1 = -10
  quotient[1] = -10

  j=0: remainder[1+0] = remainder[1] - (-10)·divisor[0]
       = -6 - (-10)·(-6) = -6 - 60 = -66

  j=1: remainder[1+1] = remainder[2] - (-10)·divisor[1]
       = -74 - (-10)·11 = -74 + 110 = 36

  j=2: remainder[1+2] = remainder[3] - (-10)·divisor[2]
       = 54 - (-10)·(-6) = 54 - 60 = -6

  j=3: remainder[1+3] = remainder[4] - (-10)·divisor[3]
       = -10 - (-10)·1 = -10 + 10 = 0  ← 최고차가 소거됨!

  remainder = [36, -66, 36, -6, 0]

━━━ i=0 (몫의 0차 계수) ━━━
  coeff = remainder[0+3] * 1 = remainder[3] * 1 = -6
  quotient[0] = -6

  j=0: remainder[0] = 36 - (-6)·(-6) = 36 - 36 = 0
  j=1: remainder[1] = -66 - (-6)·11 = -66 + 66 = 0
  j=2: remainder[2] = 36 - (-6)·(-6) = 36 - 36 = 0
  j=3: remainder[3] = -6 - (-6)·1 = -6 + 6 = 0

  remainder = [0, 0, 0, 0, 0]  ← 완전히 소거! 나누어떨어짐!

결과:
  quotient = [-6, -10]  → h(x) = -6 - 10x = -10x - 6
  remainder = [0, 0, 0, 0, 0]  → 0 (나누어떨어짐 ✓)
```

> [!note] 역순(rev) 반복의 이유
> ```
> for i in (0..=quot_deg).rev()  → i = quot_deg, quot_deg-1, ..., 0
>
> 높은 차수부터 처리하는 이유:
>   ① remainder[i + div_deg]를 소거 → 최고차 항부터 제거
>   ② 다음 반복에서는 더 낮은 차수를 처리
>   ③ 각 반복에서 정확히 하나의 차수가 소거됨
>
> 수동 long division과 정확히 같은 순서:
>   최고차 항 맞춰서 빼기 → 차수 줄어듦 → 반복
> ```

### Polynomial의 엣지 케이스와 trim()의 역할

```
영 다항식의 표현:
  coeffs = []  (빈 벡터)

  이유:
    p(x) = 0을 [0]으로 표현하면, degree가 0이 됨 (상수 0)
    p(x) = 0의 "차수"는 보통 -∞ 또는 "정의되지 않음"
    우리 코드에서는 is_zero()로 분기하므로 빈 벡터가 안전

  영향:
    Polynomial::zero() → coeffs = []
    eval(x) → 빈 벡터 → 0 반환 (early return)
    degree() → 빈 벡터 → 0 반환 (주의: 상수 0과 구분 불가)
                         → 하지만 is_zero()를 먼저 체크하면 문제 없음
```

```
trim()이 필요한 이유:

  다항식 뺄셈에서 자연스럽게 발생하는 문제:
    p(x) = x² + 3x + 2
    q(x) = x² + x + 1
    p - q = 0·x² + 2x + 1 = 2x + 1

  trim() 없이: coeffs = [1, 2, 0]  → degree = 2 (잘못!)
  trim() 후:    coeffs = [1, 2]    → degree = 1 (올바름)

코드:
    fn trim(&mut self) {
        while let Some(&c) = self.coeffs.last() {
            if c.is_zero() {
                self.coeffs.pop();     // 뒤에서부터 0 제거
            } else {
                break;                  // 비영 계수 만나면 중단
            }
        }
    }

  호출 시점: from_coeffs()에서 자동 호출
    → 사용자가 trim을 잊어도 항상 정규화된 형태 보장
```

```
엣지 케이스별 동작:

┌───────────────────────────────────┬────────────────────────┬──────────┐
│ 입력                              │ trim 후                 │ degree   │
├───────────────────────────────────┼────────────────────────┼──────────┤
│ coeffs = [0, 0, 0]               │ []                      │ 0 (zero) │
│ coeffs = [5]                     │ [5]                     │ 0        │
│ coeffs = [0, 3]                  │ [0, 3]                  │ 1        │
│ coeffs = [1, 2, 0, 0]           │ [1, 2]                  │ 1        │
│ coeffs = [0]                     │ []                      │ 0 (zero) │
│ coeffs = [0, 0, 1]              │ [0, 0, 1]               │ 2        │
└───────────────────────────────────┴────────────────────────┴──────────┘

주의: coeffs = [0, 3]은 trim하지 않는다!
  → 0은 "앞쪽" 계수 (상수항 = 0)
  → trim은 "뒤쪽" (최고차항)의 0만 제거
  → p(x) = 3x는 [0, 3]이 올바른 표현
```

```
PartialEq의 구현과 trim의 관계:

  두 다항식이 같은지 비교할 때, 길이가 다를 수 있음:
    p(x) = x + 1  → coeffs = [1, 1]
    q(x) = x + 1  → coeffs = [1, 1, 0]  (trim 전)

  코드:
    impl PartialEq for Polynomial {
        fn eq(&self, other: &Self) -> bool {
            let len = max(self.coeffs.len(), other.coeffs.len());
            for i in 0..len {
                let ca = if i < self.len() { self.coeffs[i] } else { Fr::ZERO };
                let cb = if i < other.len() { other.coeffs[i] } else { Fr::ZERO };
                if ca != cb { return false; }
            }
            true
        }
    }

  짧은 쪽은 Fr::ZERO로 패딩하여 비교
  → trim 여부와 무관하게 올바른 비교 보장
  → 하지만 trim이 되어 있으면 더 효율적 (불필요한 비교 방지)
```

> [!tip] degree()의 규약과 주의사항
> ```
> 우리 코드:
>   pub fn degree(&self) -> usize {
>       if self.coeffs.is_empty() { 0 } else { self.coeffs.len() - 1 }
>   }
>
> 주의: 영 다항식의 degree가 0을 반환함
>   수학적으로 영 다항식의 차수는 -∞ (또는 undefined)
>   하지만 Rust에서 usize는 음수 불가 → 0으로 대체
>
> 실제 사용에서 문제가 되는가?
>   → 아니다. 모든 차수 비교 전에 is_zero() 체크가 있음
>   → div_rem: if self.is_zero() → (zero, self) 조기 반환
>   → compute_h: if rem.is_zero() → Some(h)
>   → 영 다항식이 잘못된 차수를 가져도 로직에 영향 없음
>
> 프로덕션 라이브러리 (arkworks):
>   degree()가 Option<usize> 반환 — 영 다항식은 None
>   또는 Degree enum으로 NegInfinity 표현
> ```

│  ↓  항등식: a(x)·b(x) - c(x) = h(x)·t(x)                         │

│  (10x²-24x+17)(-x²+3x+1) - (-5x²+33x-19)                        │
│    = (-10x-6)(x³-6x²+11x-6)                                      │
│                                                                   │
│  QAP 만족! → Groth16 증명 생성 가능                                │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘

   올바른 witness → QAP 만족 → h(x) 존재

2. 건전성 (Soundness):
   잘못된 witness → QAP 불만족 → h(x) 부존재
   → qap_wrong_witness_fails
   → qap_cubic_wrong_output_fails

3. 보간 검증 (Interpolation):
   다항식이 도메인에서 R1CS 행렬과 일치
   → qap_polynomials_match_r1cs_at_domain

4. 항등식 검증 (Identity):
   임의의 점 τ에서 a(τ)·b(τ) - c(τ) = h(τ)·t(τ)
   → qap_identity_holds_everywhere
   → Schwartz-Zippel의 직접 적용

5. 차수 경계 (Degree Bounds):
   a,b,c ≤ m-1, h ≤ m-2, t = m
   → qap_degree_bounds
```

> [!tip] 항등식 검증 테스트의 의미
> ```
> qap_identity_holds_everywhere 테스트:
>   여러 τ 값에서 a(τ)·b(τ) - c(τ) = h(τ)·t(τ) 확인
>
> 이것이 Schwartz-Zippel의 직접 적용:
>   다항식 항등식이 여러 랜덤 점에서 성립 → 전체에서 성립
>   Groth16은 이 검증을 "암호학적으로 숨긴 τ"에서 수행
> ```

### 설계 결정과 트레이드오프

#### 결정 1: 도메인 선택

```
우리: D = {1, 2, 3, ..., m}  (단순한 점들)

  장점:
    - 구현이 단순
    - 임의의 m에 대해 즉시 사용 가능
    - 교육용으로 이해하기 쉬움

  단점:
    - Lagrange 보간이 O(n²)
    - 소거 다항식 t(x) 계수가 큼
    - FFT 불가능

프로덕션: D = {1, ω, ω², ..., ω^(m-1)}  (m-th roots of unity)

  장점:
    - NTT로 O(n log n) 보간/평가
    - t(x) = x^m - 1 — 극도로 단순
    - Groth16/PLONK에서 표준

  단점:
    - m이 2의 거듭제곱이어야 함 (또는 패딩)
    - ω 계산 필요 (Fr의 원시 m차 근)
```

#### 결정 2: 다항식 표현

```
우리: 계수 형태 (coefficient form)
  coeffs = [a₀, a₁, ..., aₙ]

  장점: 직관적, 산술 연산이 자명
  단점: 평가에 O(n), 곱셈에 O(n²)

대안: 평가 형태 (evaluation form)
  evals = [p(ω₀), p(ω₁), ..., p(ωₙ₋₁)]

  장점: 점별 곱셈이 O(n) (!!)
  단점: 계수 → 평가 변환에 FFT 필요
```

#### 결정 3: 열별 보간

```
우리: 각 열(변수)마다 독립적으로 Lagrange 보간

  n개 변수 × m개 점 → n번의 O(m²) 보간 = O(nm²)

프로덕션 최적화:
  1. 모든 0 열 스킵 (희소성 활용)
  2. Lagrange basis 재사용 (basis는 도메인에만 의존)
  3. iNTT로 O(m log m) 보간

우리 코드의 최적화:
  all_zero 체크 → 대부분의 열에서 보간 생략
  R1CS가 희소하므로 이것만으로도 상당한 절약
```

#### 결정 4: 교육 vs 프로덕션 비교

```
                    교육용 (우리)           프로덕션 (arkworks 등)
도메인            {1,2,...,m}            m-th roots of unity
보간             O(n²) Lagrange        O(n log n) iNTT
다항식 곱셈       O(n²) schoolbook      O(n log n) NTT
소거 다항식       O(m) coefficients     t(x) = x^m - 1
h(x) 계산        polynomial division   NTT + pointwise div
메모리           3nm Fr elements       on-the-fly evaluation

장단점:
  교육: 이해하기 쉬움, 코드가 직관적, 작은 회로에 적합
  프로덕션: 대규모 회로 처리 가능, 최적화된 성능
```

### 복잡도 분석: 교육용 vs 프로덕션 정량 비교

> [!important] 왜 정량 비교가 중요한가
> "교육용이라서 느린 건 괜찮다"는 말은 m이 작을 때만 유효하다.
> 아래 분석을 통해 어디까지가 "괜찮은" 범위이고,
> 어디서부터 프로덕션 최적화가 **필수**인지를 정량적으로 확인한다.

#### 연산 복잡도 비교

```
┌────────────────────────┬─────────────────────────┬──────────────────────────┐
│ 단계                    │ 교육용 (우리)            │ 프로덕션 (FFT 기반)       │
├────────────────────────┼─────────────────────────┼──────────────────────────┤
│ Lagrange 보간           │ O(n · m²)               │ O(n · m log m)  (iNTT)   │
│  (n개 변수 × m개 점)     │ 각 변수마다 O(m²) 보간   │ 각 변수마다 O(m log m)    │
├────────────────────────┼─────────────────────────┼──────────────────────────┤
│ Witness 결합            │ O(n · m)                │ O(n · m)                 │
│  a(x) = Σ sⱼ·aⱼ(x)     │ n개 다항식 스칼라곱 + 합  │ 동일                     │
├────────────────────────┼─────────────────────────┼──────────────────────────┤
│ 다항식 곱셈 a·b          │ O(m²)  (schoolbook)     │ O(m log m)  (NTT)        │
├────────────────────────┼─────────────────────────┼──────────────────────────┤
│ 다항식 나눗셈 p/t        │ O(m²)  (long division)  │ O(m log m)  (NTT 기반)   │
├────────────────────────┼─────────────────────────┼──────────────────────────┤
│ 소거 다항식 t(x) 구축    │ O(m²)                   │ O(1)  (t(x) = x^m - 1)  │
├────────────────────────┼─────────────────────────┼──────────────────────────┤
│ 전체 QAP 변환           │ O(n · m²)               │ O(n · m log m)           │
└────────────────────────┴─────────────────────────┴──────────────────────────┘
```

#### 구체적 연산 횟수 (field multiplication 기준)

```
가정: n ≈ m (변수 수 ≈ 제약 수, 일반적인 경우)

┌──────┬────────────────────────────────┬────────────────────────────────┐
│  m   │ 교육용: O(m³) field muls       │ 프로덕션: O(m² log m) field muls│
│      │ (보간 n·m² ≈ m³)               │ (보간 n·m log m ≈ m² log m)    │
├──────┼────────────────────────────────┼────────────────────────────────┤
│   10 │ 10³ = 1,000                    │ 10² × 3.3 ≈ 330               │
│      │ → 즉시 (~1ms 미만)              │ → 즉시                         │
│      │ 차이: 3배                       │                                │
├──────┼────────────────────────────────┼────────────────────────────────┤
│  100 │ 10⁶ = 1,000,000               │ 10⁴ × 6.6 ≈ 66,000            │
│      │ → ~10ms                        │ → ~1ms                         │
│      │ 차이: 15배                      │                                │
├──────┼────────────────────────────────┼────────────────────────────────┤
│ 1000 │ 10⁹ = 1,000,000,000           │ 10⁶ × 10 ≈ 10,000,000         │
│      │ → ~10초                         │ → ~100ms                       │
│      │ 차이: 100배                     │                                │
├──────┼────────────────────────────────┼────────────────────────────────┤
│10000 │ 10¹² = 1,000,000,000,000      │ 10⁸ × 13 ≈ 1,300,000,000      │
│      │ → ~2.8시간 (!!)                 │ → ~13초                        │
│      │ 차이: 769배                     │                                │
└──────┴────────────────────────────────┴────────────────────────────────┘

시간 추정 기준: BN254 Fr 곱셈 ≈ 10ns (최신 하드웨어)
```

> [!note] 실감 나는 비교
> ```
> m = 10 (우리 교육 회로):
>   교육용: 0.01ms → 전혀 문제 없음
>   결론: "느린 구현"의 불이익이 체감되지 않는다
>

> m = 1,000 (중간 규모 회로, Poseidon 등):
>   교육용: ~10초 → 불편하지만 실행은 가능

>   결론: 여기서부터 FFT의 이점이 확실해진다
>
> m = 10,000 (실용 규모 회로):
>   교육용: ~2.8시간 → 사실상 불가능
>   프로덕션: ~13초 → 실용적
>   결론: 교육용 구현은 이 규모에서 완전히 실패한다
> ```

### 다음 스텝과의 연결: Groth16

```
QAP → Groth16의 연결:

QAP가 제공하는 것:
  ① aⱼ(x), bⱼ(x), cⱼ(x): 열 다항식
  ② t(x): 소거 다항식
  ③ h(x): quotient 다항식
  ④ 항등식: a(x)·b(x) - c(x) = h(x)·t(x)

Groth16이 하는 것:
  ① Setup (trusted setup):
     비밀 τ 선택, CRS 생성:
       σ₁ = {[τⁱ]₁ : i = 0,...,m}
       σ₂ = {[τⁱ]₂ : i = 0,...,m}
       + 변수별 "키" {[aⱼ(τ)]₁, [bⱼ(τ)]₂, [cⱼ(τ)]₁}

  ② Prove:
     witness s로 다항식 평가:
       [A]₁ = Σ sⱼ·[aⱼ(τ)]₁ = [a(τ)]₁
       [B]₂ = Σ sⱼ·[bⱼ(τ)]₂ = [b(τ)]₂
       [C]₁ = ... (약간 더 복잡)

     h(x) 계산 후:
       [H]₁ = Σ hᵢ·[τⁱ]₁ = [h(τ)]₁

     증명: π = ([A]₁, [B]₂, [C]₁)  — 128 bytes

  ③ Verify:
     e([A]₁, [B]₂) = e([C]₁, [1]₂) · e([H]₁, [t(τ)]₂) · ...

     → pairing 3~4회 = O(1)
     → witness를 모르고도 검증 가능!

핵심:
  QAP의 다항식 항등식을
  타원곡선 점으로 인코딩 + 페어링으로 검증
  = Zero-Knowledge SNARK
```

> [!important] QAP가 Groth16의 전부
> ```
> Groth16 = QAP + 페어링
>
> QAP 없이는:
>   - m개 제약을 하나로 압축할 수 없음
>   - O(1) 검증이 불가능
>   - "간결한 증명"이 불가능
>
> QAP가 있으면:
>   - 다항식 항등식 1개로 환원
>   - 랜덤 τ에서 1회 검증 (Schwartz-Zippel)
>   - τ를 암호학적으로 숨김 (페어링의 역할)
>   - 128 bytes 증명, O(1) 검증 = SNARK
> ```

### ZK에서의 다항식의 의미

```
다항식이 ZK 증명의 중심인 이유:

1. 압축성 (Compression):
   m개의 값 → 1개의 degree m-1 다항식
   → "모든 제약 만족" = "다항식 나누어떨어짐"

2. 검증 효율성 (Schwartz-Zippel):
   다항식 항등식을 한 점에서 검증 → O(1)
   오류 확률: d/|F| ≈ 10^(-74)

3. Commitment 가능:
   다항식 → 타원곡선 점으로 commit
   → 값을 숨기면서 정확성 증명 가능
   → KZG commitment (Step 13)

4. 동형성 (Homomorphism):
   [p(τ)]₁ = Σ pᵢ·[τⁱ]₁
   → 다항식 산술이 타원곡선 산술로 변환
   → 이것이 Groth16과 PLONK의 핵심 기법

Schwartz-Zippel 정량 분석:
  BN254: |Fr| = 2^254 ≈ 2.9 × 10^76
  Poseidon QAP: degree ≈ 634
  → 충돌 확률: 634 / 2^254 ≈ 2.2 × 10^(-74)
  → 우주의 원자 수(10^80)보다 작은 확률

  결론: 사실상 완벽한 검증
```

### "What If" 분석: 도메인 선택의 영향

> [!abstract] 핵심 질문
> 도메인 D를 다르게 선택하면 QAP의 수학적 구조와 성능이 어떻게 변하는가?
> 여기서는 m=3 (제약 3개)일 때 네 가지 도메인 선택지를 구체적으로 비교한다.

#### 선택지 A: D = {0, 1, 2}

```
문제: 0이 도메인에 포함됨!

t(x) = (x - 0)(x - 1)(x - 2) = x(x - 1)(x - 2)
     = x³ - 3x² + 2x
     coeffs = [0, 2, -3, 1]

t(0) = 0 ← 당연히 0 (도메인 점이므로)

잠재적 문제:
  ① Lagrange 보간에서 분모에 (xᵢ - xⱼ) 계산 시:
     L₀(x) = (x-1)(x-2) / ((0-1)(0-2)) = (x-1)(x-2) / 2
     → 분모 = 2, 역원 존재 → 보간 자체는 문제 없음

  ② 하지만 evaluation point로 x=0을 사용하면:
     a(0) = Σ sⱼ·aⱼ(0) = Σ sⱼ·A[0,j]
     → 제약 0이 "x=0에서" 인코딩됨

  ③ 실질적 문제:
     - 일부 프로토콜에서 x=0은 특별한 의미 가짐
     - PLONK에서 permutation argument는 x=0을 피해야 함
     - Groth16에서는 큰 문제가 아니지만, 관행적으로 피함

  ④ t(x)의 상수항이 0:
     t(x) = x(x² - 3x + 2)
     → t(x)는 x를 인수로 가짐
     → 특정 최적화(상수항 기반 연산)에서 문제 가능
```

```
수동 계산: D = {0, 1, 2}에서의 Lagrange 기저

L₀(x) = (x-1)(x-2) / ((0-1)(0-2)) = (x²-3x+2)/2
L₁(x) = (x-0)(x-2) / ((1-0)(1-2)) = x(x-2)/(-1) = -x²+2x
L₂(x) = (x-0)(x-1) / ((2-0)(2-1)) = x(x-1)/2   = (x²-x)/2

검증:
  L₀(0)=1, L₀(1)=0, L₀(2)=0 ✓
  L₁(0)=0, L₁(1)=1, L₁(2)=0 ✓
  L₂(0)=0, L₂(1)=0, L₂(2)=1 ✓

→ 수학적으로 작동하지만, 0 포함의 관행적/구조적 문제 존재
```

#### 선택지 B: D = {1, 2, 3} (우리의 선택)

```
t(x) = (x-1)(x-2)(x-3) = x³ - 6x² + 11x - 6
coeffs = [-6, 11, -6, 1]

장점:
  ① 단순함: Fr::from_u64(1), Fr::from_u64(2), ... 로 생성
  ② 0을 포함하지 않아 잠재적 문제 회피
  ③ 교육용으로 계산 추적이 용이 (작은 정수)
  ④ 모든 도메인 점 간 차이가 non-zero → 역원 항상 존재

단점:
  ① FFT 사용 불가 (roots of unity가 아님)
  ② t(x) 계수가 일반적인 정수 → 대규모에서 비효율
  ③ Lagrange 보간이 O(m²) — 가속 방법 없음

m이 커질 때의 문제:
  m = 100일 때: t(x)의 계수들이 매우 큰 수
  (x-1)(x-2)...(x-100)의 전개 → 계수가 100! 규모
  → Fr에서는 mod p로 표현되므로 overflow 문제는 없으나
    "구조적 단순함"이 사라짐
```

#### 선택지 C: D = {ω, ω², ω³} (roots of unity, ω³=1)

```
ω = primitive 3rd root of unity in Fr
  ω³ = 1,  ω ≠ 1
  ω² + ω + 1 = 0  (최소다항식)

BN254에서:
  ω₃ = Fr의 3차 원시근 (존재함, |Fr|-1 = r-1이 3으로 나누어짐)

t(x) = (x - ω)(x - ω²)(x - ω³)
      = (x - ω)(x - ω²)(x - 1)

핵심: ω, ω², ω³=1 은 x³ = 1의 세 근
  → t(x) = x³ - 1  (!!)

         ┌─────────────────────────────────────────┐
         │  t(x) = x³ - 1                           │
         │  coeffs = [-1, 0, 0, 1]                   │
         │                                           │
         │  비교:                                     │
         │  우리:     [-6, 11, -6, 1]  (4개 비영 계수)  │
         │  RoU:      [-1,  0,  0, 1]  (2개 비영 계수!) │
         │                                           │
         │  일반화: m개 제약 → t(x) = x^m - 1          │
         │    → 항상 2개의 비영 계수만 존재              │
         │    → O(1) 저장, O(log m) 평가              │
         └─────────────────────────────────────────┘

왜 x^m - 1인가?

  증명:
    ω가 primitive m-th root of unity이면:
      ω^m = 1
      {ω, ω², ..., ω^m} = {x : x^m = 1}의 모든 근

    x^m - 1 = ∏_{i=1}^{m} (x - ω^i)   (인수정리)

    → 정의에 의해 t(x) = ∏(x - ω^i) = x^m - 1

  이것이 roots of unity가 QAP에서 "마법"인 이유:
    임의 도메인: t(x) 구축에 O(m²) 곱셈, m+1개 계수 저장
    RoU 도메인:  t(x) = x^m - 1, O(1) 구축, 2개 계수 저장
```

```
NTT (Number Theoretic Transform) 가속:

roots of unity의 추가 이점:
  다항식 평가/보간이 FFT로 가속됨

  ┌──────────────────────────────────────────────────────┐
  │  계수 형태          NTT (O(m log m))          평가 형태  │
  │  [a₀, a₁, ..., aₘ] ──────────────→ [p(ω), p(ω²), ...] │
  │                     ←──────────────                     │
  │                    iNTT (O(m log m))                    │
  │                                                         │
  │  비교:                                                   │
  │    우리: 평가 O(m²) Lagrange 보간                         │
  │    NTT:  평가 O(m log m)                                 │
  │                                                         │
  │    m=1000:  우리 10⁶ vs NTT 10⁴  → 100배 차이            │
  │    m=10⁶:  우리 10¹² vs NTT 2×10⁷ → 50,000배 차이       │
  └──────────────────────────────────────────────────────┘

NTT의 원리 (butterfly algorithm):
  DFT와 동일한 구조, 체 F_p 위에서 수행
  ω^(m/2) = -1 성질을 활용하여 재귀적 분할

  NTT([a₀,...,aₘ₋₁]) = butterfly 연산 O(m log m)번
```

#### 선택지 D: D = {random points}

```
D = {r₁, r₂, r₃}  (Fr에서 랜덤 선택)

예: r₁ = 17, r₂ = 42, r₃ = 99

t(x) = (x - 17)(x - 42)(x - 99)

장점:
  ① 수학적으로 완전히 올바름 — Lagrange 보간, 소거 다항식 모두 작동
  ② 도메인 점이 서로 다르기만 하면 됨 (모든 차이의 역원 존재)

단점:
  ① FFT 불가능 — 구조적 성질 없음
  ② t(x) 계수가 예측 불가 — 디버깅 어려움
  ③ 재현성 문제 — 같은 회로라도 도메인이 다르면 다른 QAP
  ④ 보안 우려: 도메인이 secret이면 문제 없지만,
     public이면 특정 도메인이 더 "취약"할 가능성은?
     → Schwartz-Zippel에 의해 도메인 자체의 선택은 보안에 영향 없음
     → 다만, t(τ)의 계산 효율성에 영향

  ⑤ t(x) 계수 예:
     (x-17)(x-42) = x² - 59x + 714
     (x² - 59x + 714)(x - 99) = x³ - 158x² + 6555x - 70686
     coeffs = [-70686, 6555, -158, 1]  — 큰 계수!
```

#### 네 가지 선택지 비교 요약

```
┌──────────────────┬───────────────┬────────────────┬─────────────────┬───────────────┐
│                  │ D={0,1,2}     │ D={1,2,3}      │ D={ω,ω²,ω³}    │ D={r₁,r₂,r₃} │
├──────────────────┼───────────────┼────────────────┼─────────────────┼───────────────┤
│ t(x)             │ x³-3x²+2x    │ x³-6x²+11x-6  │ x³-1            │ 큰 계수        │
│ 비영 계수 수      │ 3             │ 4              │ 2               │ 4             │
│ FFT 가능?         │ ✗             │ ✗              │ ✓               │ ✗             │
│ 0 포함?           │ ✓ (문제)      │ ✗              │ ✗               │ 아마 ✗        │
│ 보간 복잡도       │ O(m²)         │ O(m²)          │ O(m log m)      │ O(m²)         │
│ t(x) 평가        │ O(m)          │ O(m)           │ O(log m)        │ O(m)          │
│ 교육 적합성       │ 보통           │ 높음           │ 낮음 (추상적)    │ 낮음          │
│ 프로덕션 적합성   │ 낮음           │ 낮음           │ 높음 (표준)      │ 매우 낮음      │
└──────────────────┴───────────────┴────────────────┴─────────────────┴───────────────┘
```

> [!tip] 결론: 도메인 선택은 "정확성"이 아니라 "효율성"의 문제
> 어떤 도메인을 선택하든 QAP의 수학적 정확성은 동일하다.
> 모든 점이 서로 다르기만 하면 Lagrange 보간이 유일한 다항식을 생성하고,
> t(x)가 올바른 소거 다항식이 된다.
>
> 차이는 **성능**에서 나타난다:
> - roots of unity → t(x) = x^m - 1, FFT 가속, 프로덕션 표준
> - 연속 정수 → 단순하고 교육적이지만 O(m²) 보간
> - 랜덤 점 → 수학적으로 유효하지만 실용적 이점 없음
