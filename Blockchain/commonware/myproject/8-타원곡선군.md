## Step 05-1: G1 — BN254 위의 타원곡선 군

### 타원곡선이란?

```
유한체 Fp 위에서 y² = x³ + ax + b 를 만족하는 점 (x, y)의 집합 + 무한원점 O

BN254: a = 0, b = 3
  → y² = x³ + 3
```

> [!important] 왜 타원곡선인가? 타원곡선 위의 점들은 **군(group)**을 이룬다:
> 
> - 두 점을 "더해서" 새 점을 만들 수 있고
> - 항등원(무한원점 O)이 있고
> - 역원(-P)이 있다
> 
> 이 군에서 `k·P`(스칼라 곱)는 쉽지만, `P`와 `Q = k·P`로부터 `k`를 구하는 것은 어렵다. → **이산로그 문제(ECDLP)** → 암호학적 안전성의 기반

---

### 좌표 시스템: Affine vs Jacobian

#### Affine 좌표 (x, y)

직관적이지만, **점 덧셈마다 역원이 필요**하다:

```
P₁ + P₂의 기울기: λ = (y₂ - y₁) / (x₂ - x₁)
                                      ^^^^^^^^^
                                      Fp::inv 필요!
```

Fp::inv = Fermat 소정리 `a^(p-2)` = **254번의 square + mul** → 매우 비쌈

#### Jacobian 좌표 (X, Y, Z)

```
Affine ↔ Jacobian 변환:
  affine (x, y) = (X/Z², Y/Z³)

  Affine → Jacobian: (x, y) → (x, y, 1)     ← 무료
  Jacobian → Affine: (X, Y, Z) → (X/Z², Y/Z³) ← inv 1번
```

> [!tip] Jacobian의 장점 덧셈/더블링을 **역원 없이** Fp의 mul/add만으로 수행. 최종 결과가 필요할 때만 `to_affine`으로 **역원 1번** 호출.
> 
> 페어링의 Miller loop에서 수백 번의 덧셈/더블링이 일어나므로, 매번 inv를 피하는 것이 **극적인** 성능 차이를 만든다.

### 생성자와 항등원

```rust
impl G1 {
    /// 항등원 (무한원점): Z = 0인 점
    pub fn identity() -> Self {
        G1 { x: Fp::ZERO, y: Fp::ONE, z: Fp::ZERO }

        G1 {

            y: Fp::from_u64(2),
            z: Fp::ONE,
        }
    }

```

> [!note] BN254의 생성자가 (1, 2)인 이유 y² = x³ + 3 에 x=1 대입하면 y² = 4, y = 2. 이 점이 위수 r인 부분군을 생성하는지는 별도 검증이 필요하지만, BN254에서는 이것이 표준 생성자.

### 커브 위 확인

```
Affine:   y² = x³ + b
Jacobian: Y² = X³ + b·Z⁶    ← (X/Z², Y/Z³)을 대입하면 유도됨
```

```rust
/// Jacobian 좌표에서 커브 위 확인: Y² = X³ + b·Z⁶
pub fn is_on_curve(&self) -> bool {
    if self.is_identity() { return true; }
    let b = Fp::from_u64(3);
    let z2 = self.z.square();
    let z6 = z2.square() * z2;    // Z⁴ · Z² = Z⁶
    self.y.square() == self.x.square() * self.x + b * z6
}
```

### 점 더블링: 2P

기하학적 의미: 곡선 위의 점 P에서 **접선**을 그어 곡선과 만나는 점을 x축 대칭.

```
a=0 특화 공식 (BN254):

A = Y²
B = 4·X·A           → 4XY²
C = 8·A²            → 8Y⁴
D = 3·X²            → 3X² (a=0이므로 +aZ⁴ 항 없음)

X₃ = D² - 2B
Y₃ = D·(B - X₃) - C
Z₃ = 2·Y·Z
```

```rust
pub fn double(&self) -> G1 {
    if self.is_identity() || self.y.is_zero() {
        return G1::identity();
    }

    let two = Fp::from_u64(2);
    let three = Fp::from_u64(3);
    let four = Fp::from_u64(4);
    let eight = Fp::from_u64(8);

    let a = self.y.square();
    let b = four * self.x * a;          // 4·X·Y²
    let c = eight * a.square();          // 8·Y⁴
    let d = three * self.x.square();     // 3·X²

    let x3 = d.square() - two * b;
    let y3 = d * (b - x3) - c;
    let z3 = two * self.y * self.z;

    G1 { x: x3, y: y3, z: z3 }
}
```

> [!note] 역원 호출 0회 Affine 더블링은 `λ = 3x²/(2y)` → **inv 1회** 필요. Jacobian은 mul/square만으로 완료. 이것이 Jacobian의 존재 이유.

### 점 덧셈: P + Q

기하학적 의미: 두 점 P, Q를 잇는 **직선**이 곡선과 만나는 세 번째 점을 x축 대칭.

```
U₁ = X₁·Z₂²    U₂ = X₂·Z₁²
S₁ = Y₁·Z₂³    S₂ = Y₂·Z₁³
H = U₂ - U₁    R = S₂ - S₁

X₃ = R² - H³ - 2·U₁·H²
Y₃ = R·(U₁·H² - X₃) - S₁·H³
Z₃ = H·Z₁·Z₂
```

엣지 케이스:

```
U₁ == U₂ && S₁ == S₂  →  P == Q  →  double(P) 호출
U₁ == U₂ && S₁ != S₂  →  P == -Q →  identity 반환
```

```rust
pub fn add(&self, rhs: &G1) -> G1 {
    if self.is_identity() { return *rhs; }
    if rhs.is_identity() { return *self; }

    let z1s = self.z.square();
    let z2s = rhs.z.square();
    let u1 = self.x * z2s;
    let u2 = rhs.x * z1s;
    let s1 = self.y * z2s * rhs.z;
    let s2 = rhs.y * z1s * self.z;

    if u1 == u2 {
        if s1 == s2 { return self.double(); }  // P == Q
        else { return G1::identity(); }          // P == -Q
    }

    let h = u2 - u1;
    let r = s2 - s1;
    let h2 = h.square();
    let h3 = h * h2;
    let u1h2 = u1 * h2;

    let x3 = r.square() - h3 - Fp::from_u64(2) * u1h2;
    let y3 = r * (u1h2 - x3) - s1 * h3;
    let z3 = h * self.z * rhs.z;

    G1 { x: x3, y: y3, z: z3 }
}
```

### 스칼라 곱: k·P

```
k = 이진수 b₂₅₃b₂₅₂...b₁b₀

알고리즘 (double-and-add):
  result = O (항등원)
  base = P
  for each bit bᵢ:
    if bᵢ == 1: result = result + base
    base = 2·base
```

```rust
pub fn scalar_mul(&self, scalar: &[u64; 4]) -> G1 {
    let mut result = G1::identity();
    let mut base = *self;
    for &limb in scalar.iter() {
        for j in 0..64 {
            if (limb >> j) & 1 == 1 {
                result = result.add(&base);
            }
            base = base.double();
        }
    }
    result
}
```

> [!tip] 복잡도 254-bit 스칼라: 최대 **254 doublings + 254 additions** 이것이 타원곡선 암호의 기본 연산이고, 페어링의 Miller loop도 이 패턴의 변형.

### Jacobian → Affine 변환

**이 순간에만** 역원을 1번 호출:

```rust
pub fn to_affine(&self) -> G1Affine {
    if self.is_identity() {
        return G1Affine::identity();
    }
    let z_inv = self.z.inv().unwrap();     // ← 유일한 inv
    let z_inv2 = z_inv.square();
    let z_inv3 = z_inv2 * z_inv;
    G1Affine {
        x: self.x * z_inv2,    // X/Z²
        y: self.y * z_inv3,    // Y/Z³
        infinity: false,
    }
}
```

### Jacobian 동치 비교

같은 Affine 점이 **다른 Jacobian 표현**을 가질 수 있다:

```
(X, Y, Z) 과 (λ²X, λ³Y, λZ) 는 같은 점
```

to_affine 없이 비교하려면:

```rust
impl PartialEq for G1 {
    fn eq(&self, other: &Self) -> bool {
        // X₁·Z₂² == X₂·Z₁² and Y₁·Z₂³ == Y₂·Z₁³
        let z1s = self.z.square();
        let z2s = other.z.square();
        self.x * z2s == other.x * z1s
            && self.y * z2s * other.z == other.y * z1s * self.z
    }
}
```

### 테스트: 군 공리 + 위수 검증

```rust
#[test]
fn scalar_mul_order_gives_identity() {
    // r·G = O — G1의 위수가 r임을 검증
    let g = G1::generator();
    let r = [
        0x43e1f593f0000001, 0x2833e84879b97091,
        0xb85045b68181585d, 0x30644e72e131a029,
    ];
    assert_eq!(g.scalar_mul(&r), G1::identity());
}

#[test]
fn scalar_mul_distributive() {
    // 5G = 2G + 3G
    let g = G1::generator();
    assert_eq!(
        g.scalar_mul(&[5, 0, 0, 0]),
        g.scalar_mul(&[2, 0, 0, 0]) + g.scalar_mul(&[3, 0, 0, 0])
    );
}
```

> [!abstract] r·G = O 의 의미 G1의 모든 점은 위수가 r (= Fr의 modulus). 생성자 G를 r번 더하면 원점으로 돌아온다. 이것은 **시계가 12에서 0으로 돌아가는 것**과 같은 원리. 이 r이 바로 ZK에서 witness가 사는 **Fr 체의 크기**.

### 전체 구조에서 G1의 위치

```
Fp (좌표)  ──→  G1 점의 좌표           ← 이번 스탭
Fr (스칼라) ──→  G1의 scalar_mul 입력
Fp2        ──→  G2 점의 좌표           ← 다음 스탭
Fp12       ──→  페어링 결과 e(G1, G2)

Groth16 검증식:
  e(A, B) == e(α, β) · e(L, γ) · e(C, δ)
  ^^^        ^^^       ^^^       ^^^
  G1×G2      G1×G2     G1×G2     G1×G2
```
